# ARANGEMENT VISION — Стиль как главный принцип организации аранжировки

**Дата:** 2026-07-01
**Горизонт:** 1 версия (4–6 недель)
**Статус:** 🟡 Черновик

## 1. Резюме (Executive Summary)

Рефакторинг подсистемы аранжировки Jazz Trainer с переворотом логики организации: **не инструмент → стиль, а стиль → инструменты**. Сейчас стиль — это атрибут, «прикрученный» к инструментам сверху. Каждый инструмент знает про стили, но стиль не знает про инструменты. Это создаёт рассогласованность: пользователь выбирает стиль swing, но не понимает, какой набор инструментов для него релевантен, какие настройки характерны, и почему при смене стиля на bossa гитара всё ещё играет steel-string, а не nylon.

Новый принцип: **стиль — главенствующая сущность**. Каждый стиль определяет:
- **Состав ансамбля** (какие инструменты доступны, какие обязательны, какие опциональны)
- **Настройки инструментов по умолчанию** (voicing, паттерн, громкость, регистр)
- **Характерный уровень свинга** (swing ratio)
- **Дефолтный темп** (BPM)
- **Паттерны игры** для каждого инструмента в этом стиле
- **Новые инструменты**, характерные именно для этого стиля

Пользователь выбирает стиль — и получает готовый ансамбль с преднастроенными инструментами, который можно тонко дотюнить. Это снижает когнитивную нагрузку («я хочу bossa nova — дайте мне всё для bossa nova») и открывает пространство для стилистически точных аранжировок.

**Ключевая ценность:** Jazz Trainer становится не просто «тренажёром с набором инструментов», а **стилистически осмысленным аранжировщиком**, где каждый стиль звучит аутентично благодаря правильному подбору инструментов, их настроек и паттернов.

## 2. Конкурентный контекст

| Конкурент | Организация стилей | Инструменты | Стиле-специфичные ансамбли |
|---|---|---|---|
| **iReal Pro** | Стиль выбирается глобально, влияет на MIDI-паттерны | Фортепиано, бас, барабаны (MIDI) — фиксированный набор для всех стилей | Нет — один и тот же набор для всех стилей |
| **Band-in-a-Box** | Стиль = набор MIDI-паттернов для каждого инструмента | Десятки инструментов, но стиль — это просто «пакет паттернов», не определяет состав | Частично: стиль может предложить другие инструменты, но не структурировано |
| **GarageBand / Logic Drummer** | Виртуальный барабанщик с характером и стилем | Только барабаны | Нет — только один тип инструмента |
| **Jazz Trainer (сейчас)** | Стиль — глобальный атрибут, не структурирует инструменты | 6 инструментов, все доступны всегда | Нет — пользователь сам собирает ансамбль |
| **Jazz Trainer (цель)** | Стиль — главная сущность; определяет ростер инструментов, их настройки, свинг, темп | 15+ инструментов; для каждого стиля — свой релевантный набор | Да — каждый стиль даёт аутентичный ансамбль |

**Ниша после рефакторинга:** Jazz Trainer — единственный веб-инструмент, где стиль **структурирует** ансамбль, а не просто переключает паттерны. Пользователь получает не «пианино + бас + барабаны и выбери им стиль», а «Swing-ансамбль: фортепиано, контрабас, джазовые барабаны, Rhodes-подушка, опционально — труба и вибрафон». Это ближе к реальной музыкальной практике, где каждый стиль диктует свой инструментарий.

## 3. Новые функции (Features)

### 3.1. Стиль как главенствующая сущность (Style-driven Architecture)

- **Проблема:** Сейчас `Style` — это просто enum (`'swing' | 'bossa' | 'funk' | 'latin' | 'ballad'`), который передаётся в инструменты. Инструменты сами решают, что с ним делать. Нет централизованного знания: «какие инструменты нужны для этого стиля», «какой свинг характерен», «какой темп по умолчанию». Пользователь вынужден экспериментировать с инструментами вслепую.
- **Описание:** Стиль становится объектом первого класса — `StyleProfile`. Каждый `StyleProfile`:
  ```ts
  interface StyleProfile {
    id: Style;                          // 'swing' | 'bossa' | 'funk' | 'latin' | 'ballad'
    name: string;                       // Человекочитаемое: 'Swing', 'Bossa Nova', ...
    description: string;                // Краткое описание стиля для UI

    // Ростер инструментов
    instrumentRoster: InstrumentRoster;

    // Характеристики стиля
    defaultTempo: number;               // BPM по умолчанию
    swingRatio: number;                 // 0.0 (straight) – 0.85 (heavy swing)
    timeSignaturePresets: TimeSignature[]; // характерные размеры: 4/4 для swing, 2/4 для bossa

    // Настройки инструментов по умолчанию
    instrumentDefaults: Record<InstrumentId, InstrumentStyleDefaults>;
  }

  interface InstrumentRoster {
    /** Инструменты, без которых стиль не звучит */
    required: InstrumentId[];
    /** Рекомендованные дополнительные инструменты */
    recommended: InstrumentId[];
    /** Инструменты, доступные опционально (можно включить вручную) */
    optional: InstrumentId[];
    /** Инструменты, скрытые для этого стиля (не появляются в UI) */
    hidden: InstrumentId[];
  }

  interface InstrumentStyleDefaults {
    enabled: boolean;
    volume: number;
    /** Стиле-специфичный паттерн/профиль */
    pattern?: string;
    /** Стиле-специфичный voicing */
    voicing?: VoicingDensity;
    /** Стиле-специфичный режим (для Rhodes — layer mode) */
    mode?: string;
  }
  ```

  **5 стилей и их профили:**

  | Стиль | Дефолтный темп | Свинг | Характерные размеры |
  |---|---|---|---|
  | Swing | 140 BPM | 0.67 (классический триольный свинг) | 4/4, 3/4 |
  | Bossa Nova | 120 BPM | 0.0 (straight) | 2/4, 4/4 |
  | Funk | 100 BPM | 0.12 (лёгкий свинг) | 4/4 |
  | Latin | 160 BPM | 0.0 (straight) | 4/4, 6/8 |
  | Ballad | 60 BPM | 0.58 (мягкий свинг) | 4/4, 3/4 |

- **Конкуренты:** iReal Pro — стиль влияет на паттерны, но не на состав; Band-in-a-Box — частично: стиль может менять инструменты, но это не структурировано; **никто не даёт style-driven ансамбль с явным ростером.**
- **Приоритет:** P0 (MVP)
- **Усиливает / Новая категория:** `music-core` (StyleProfile, StyleRegistry), `core-player` (Style Selector UI)
- **Зависимости:** Все остальные функции этой версии зависят от 3.1.

### 3.2. Стиле-специфичные ростеры инструментов

- **Проблема:** Все 6 инструментов видны всегда. Пользователь в стиле bossa видит Guitar со steel-string звуком и не понимает, что для bossa характерна nylon-гитара. В стиле funk пользователь не видит electric bass, хотя это ключевой инструмент жанра.
- **Описание:** Для каждого стиля — свой ростер. При выборе стиля UI адаптируется: показывает только релевантные инструменты, подсвечивает рекомендованные, скрывает нерелевантные (но позволяет открыть «расширенные настройки» для ручного включения).

  #### 3.2.1. Ростер: Swing (Mainstream Jazz)

  | Роль | Инструмент | Статус | Сэмплы / источник | Характерный паттерн |

  | 🥁 Drums | Jazz Kit (Swirly Drums v2) | **required** | Swirly Drums v2 | Ride-heavy swing pattern, hi-hat chick на 2 и 4 |
  | 🎸 Bass | Upright Bass | **required** | Upright Bass, pluck/mute ×4 RR | Walking bass (четверти + approach notes) |
  | 🎹 Piano | Grand Piano | **required** | Grand Piano | swing-sparse или swing-medium profile |
  | 🎛️ Rhodes | Rhodes | **recommended** | Rhodes, 4 velocity-слоя | subtle-offbeats — лёгкие акценты на 2&, 4& |
  | 🎺 Trumpet | Muted Trumpet (новый) | **optional** | [assumption:要找 или синтезировать] | Мелодическая линия / акценты на сильные доли |
  | 🎵 Vibraphone | Vibraphone (новый) | **optional** | [assumption: сэмплы вибрафона] | Мягкие аккордовые подклады, соло-вставки |

- **Усиливает / Новая категория:** `music-core` (StyleRegistry, InstrumentRoster), `core-player` (Roster UI)
- **Зависимости:** 3.1 (StyleProfile)

### 3.3. Новые инструменты (8 шт.)

Новые инструменты, отсутствующие в текущей версии. Добавляются пакетно — по мере готовности сэмплов. Приоритет внутри группы определяется стилем, для которого инструмент ключевой.

#### 3.3.1. Modern Kit (P0)

- **Проблема:** Swirly Drums v2 — акустическая джазовая установка с мягким, округлым звуком. В funk и latin нужны более резкие, punchy звуки: tight kick, crisp snare, sharp hi-hat. Текущие сэмплы не дают нужной атаки.
- **Описание:** Второй набор барабанных сэмплов — «Modern Kit». Те же 8 звуков (bassDrum, snare, hihat, hihatHalf, hihatOpen, ride, crash, rim), но с более агрессивным, современным характером. `DrumInstrument` уже поддерживает переключение манифестов → можно переиспользовать существующий код, заменив только `SampleManifest`.
- **Приоритет:** P0 — ключевой для funk и latin-стилей
- **Слой:** `music-core` (новый `modernKitManifest.ts`), сэмплы в `apps/web/public/samples/`

#### 3.3.2. Electric Bass (P0)

- **Проблема:** Upright Bass — сэмплы акустического контрабаса. В funk, latin и некоторых swing-контекстах нужен электрический бас (fingerstyle / slap).
- **Описание:** Новый `ElectricBassInstrument` или вариант `BassInstrument` с другим `SampleManifest`. Те же паттерны (walking, root-5th, syncopated, montuno, two-feel), но другой тембр.
- **Приоритет:** P0 — ключевой для funk
- **Слой:** `music-core`

#### 3.3.3. Nylon Guitar (P1)

- **Проблема:** Текущая `GuitarInstrument` поддерживает nylon и steel, но nylon не оптимизирован под bossa-компинг (характерный ритмический рисунок с басовой нотой + аккордами).
- **Описание:** Доработка `GuitarInstrument`: новый стиле-специфичный паттерн `bossa-comping` для nylon-гитары — характерный ритм с чередованием басовой ноты и аккорда. Nylon-гитара становится recommended для bossa.
- **Приоритет:** P1 — определяет звучание bossa
- **Слой:** `music-core` (guitarManifest, guitarInstrument)

#### 3.3.4. Electric Guitar (P1)

- **Проблема:** Для funk нужна электрогитара с funk-chops (резкие аккорды на offbeat).
- **Описание:** Новый `SampleManifest` для электрогитары и новый паттерн `funk-chops`. Steel-string остаётся для swing (Freddie Green comping на четверти).
- **Приоритет:** P1 — определяет звучание funk
- **Слой:** `music-core`

#### 3.3.5. Trumpet (P2)

- **Проблема:** Нет мелодического инструмента для духовой импровизации. Труба — ключевой голос в джазе и фанке.
- **Описание:** Новый `TrumpetInstrument` (одноголосный, pitched). Два режима: muted (для swing — мягкий, приглушённый тембр) и open (для funk/latin — яркий, открытый). Паттерны: мелодическая импровизация по гамме аккорда, ритмические акценты, долгие ноты.
- **Приоритет:** P2
- **Слой:** `music-core` (trumpetInstrument.ts, trumpetManifest.ts)

#### 3.3.6. Vibraphone (P2)

- **Проблема:** Нет инструмента с «кристальным», продолжительным звучанием для баллад и ambient-текстур.
- **Описание:** Новый `VibraphoneInstrument` (полифонический, pitched). Характерное вибрато (LFO на амплитуду/частоту), мягкая атака. Паттерны: аккордовые подклады (pads), мелодические вставки.
- **Приоритет:** P2
- **Слой:** `music-core`

#### 3.3.7. Flute (P2)

- **Проблема:** Нет воздушного, «дышащего» голоса для bossa, latin и баллад.
- **Описание:** Новый `FluteInstrument` (одноголосный, pitched). Мягкий тембр, естественное вибрато. Паттерны: мелодические пассажи, воздушные фразы.
- **Приоритет:** P2
- **Слой:** `music-core`

#### 3.3.8. Organ (Hammond-style) (P2)

- **Проблема:** Нет плотного, «греющего» гармонического инструмента для funk и soul-jazz.
- **Описание:** Новый `OrganInstrument` (полифонический, pitched). Характерный «тёплый» тембр с возможностью добавления перкуссионного клика и Leslie-эффекта. Паттерны: плотные аккордовые подклады, ритмические stabs.
- **Приоритет:** P2
- **Слой:** `music-core`

#### 3.3.9. Percussion Kit (P1)

- **Проблема:** Сейчас в bossa ритм-секция — это джазовые барабаны с rim/clave. Для полноценного латиноамериканского звучания нужны отдельные перкуссионные инструменты.
- **Описание:** Новый `PercussionInstrument` (unpitched, как `DrumInstrument`). Звуки: congas (high/low), timbales, cowbell, clave, shaker, guiro, triangle. Паттерны: cascara, clave (son/rumba), tumbao, montuno-ритм.
- **Приоритет:** P1 — определяет звучание latin-стиля
- **Слой:** `music-core` (percussionInstrument.ts, percussionManifest.ts)

#### 3.3.10. Clarinet (P3)

- **Проблема:** Нет деревянного духового для диксиленда, свинга и камерных составов.
- **Описание:** Новый `ClarinetInstrument` (одноголосный, pitched). Мягкий, «деревянный» тембр. Паттерны: контрапункт, мелодические фразы.
- **Приоритет:** P3 — nice-to-have
- **Слой:** `music-core`

### 3.4. Стиле-специфичные voicing-профили для Piano и Rhodes

- **Проблема:** Сейчас пользователь вручную выбирает voicing density (`shell2`, `rootless3`, `rootless4`, `quartal`) независимо от стиля. Но разные стили предполагают разные voicing'и: quartal аккорды характерны для latin/modal jazz, rootless4 — для плотного swing, shell2 — для bossa.
- **Описание:** Каждый `StyleProfile.instrumentDefaults` задаёт **стиле-специфичный voicing по умолчанию**:

  | Стиль | Piano Voicing по умолчанию | Rhodes Voicing по умолчанию | Обоснование |
  |---|---|---|---|
  | **Swing** | `rootless3` | `rootless3` | Классический джазовый voicing — 3, 7 и color tone. Оставляет место для баса |
  | **Bossa Nova** | `shell2` | `shell2` | Более открытое, воздушное звучание. Не перегружает ритмически плотный bossa-груз |
  | **Funk** | `rootless4` | `rootless4` | Плотное, перкуссивное звучание. 4 голоса дают насыщенный аккорд под синкопированный грув |
  | **Latin** | `quartal` | `rootless3` | Квартовые стеки — классика montuno/salsa piano (McCoy Tyner, Eddie Palmieri) |
  | **Ballad** | `rootless4` | `shell2` | Piano — богатое, полное звучание. Rhodes-слой — тонкий, не перебивает |

- Пользователь может переопределить voicing вручную в настройках — но дефолт даёт аутентичное звучание.
- **Конкуренты:** Никто не даёт стиле-специфичные voicing'и.
- **Приоритет:** P1
- **Усиливает:** `music-core` (StyleProfile.instrumentDefaults)
- **Зависимости:** 3.1, 3.2

### 3.5. Стиле-специфичные паттерны игры

- **Проблема:** Сейчас выбор паттерна для каждого инструмента — ручной. Пользователь не знает, какой паттерн характерен для стиля: например, что в swing бас должен играть walking bass, а не montuno.
- **Описание:** Каждый `StyleProfile.instrumentDefaults` задаёт **стиле-специфичный паттерн/профиль/режим по умолчанию**:

  | Инструмент | Swing | Bossa | Funk | Latin | Ballad |
  |---|---|---|---|---|---|
  | **Drums** | ride-heavy swing pattern | bossa (rim/clave) | funky 16th groove | modern kit + percussion (cascara/clave) | brushes swing (низкая громкость) |
  | **Bass** | walking bass (четверти + approach) | root-5th (половинные) | syncopated eighths (пропуск 1-й доли) | montuno (нота-пауза-нота) | two-feel (половинные) |
  | **Piano** | swing-sparse / swing-medium profile | swing-sparse profile | offbeat-push profile | basie-light / montuno-компинг | beginner-safe profile |
  | **Rhodes** | subtle-offbeats (2&, 4&) | ambient-swells (раз в 2–4 такта) | stab-accents (2 и 4) | high-comping (+12 полутонов) | pads (целые ноты) |
  | **Guitar (nylon)** | — | bossa-comping (бас + аккорд) | — | — | — |
  | **Guitar (electric)** | — | — | funk-chops (offbeat) | — | — |
  | **Guitar (steel)** | Freddie Green comping (четверти, C3–C4) | — | — | — | — |
  | **Trumpet** | muted, мелодические фразы | — | open, синкопированные акценты | open, мелодическая линия | — |
  | **Flute** | — | воздушные пассажи | — | латино-пассажи | мягкие, воздушные фразы |
  | **Vibraphone** | мягкие аккордовые подклады | — | — | мелодические вставки | продолжительные аккорды с вибрато |
  | **Organ** | — | — | плотные подклады, stabs | — | — |

- **Приоритет:** P1
- **Усиливает:** `music-core` (StyleProfile.instrumentDefaults), каждый `*Instrument`
- **Зависимости:** 3.1, 3.2, 3.3

### 3.6. Реорганизация области настроек (Settings UX)

- **Проблема:** Сейчас настройки — плоский список: `drumsEnabled`, `drumsVolume`, `drumsPattern`, `bassEnabled`, `bassVolume`, ... — все инструменты вперемешку. С добавлением 8 новых инструментов плоский список станет неюзабельным. Пользователь теряется: «какие настройки важны для bossa, а какие — для funk?»
- **Описание:** Новая структура настроек — страница с четырьмя зонами:

  **Зона 1 — Раздел «Стили» (верх страницы):**

  ```
  ┌─ Стили ─────────────────────────────────────────────┐
  │                                                       │
  │  [ Swing ]  [ Bossa Nova ]  [ Funk ]  [ Latin ]  [ Ballad ]  │
  │                                                       │
  └───────────────────────────────────────────────────────┘
  ```

  Стиль выбирается **кнопками-тогглами** (как переключатель темы), а не выпадающим списком. Один клик — смена стиля. Активный стиль подсвечен. Все 5 стилей видны сразу — не нужно раскрывать меню.

  **Зона 2 — Плитка стиля (темп и свинг):**

  ```
  ┌─ Swing ─────────────────────────────────────────────┐
  │                                                       │
  │  Темп:   [ 140 BPM ]  (↙ сбросить на дефолт стиля)  │
  │  Свинг:  [●●●●○] 0.67  (↙ сбросить на дефолт стиля) │
  │                                                       │
  │  Ансамбли:  [ Трио ]  [ Квартет ]  [ Квинтет ]  [ Сбросить ]  │
  │                                                       │
  └───────────────────────────────────────────────────────┘
  ```

  Темп и свинг — главные глобальные настройки стиля, всегда перед глазами. Рядом — кнопка «сбросить на дефолт стиля» (↺), если пользователь ушёл далеко от стилевого значения и хочет вернуться.

  **Ансамбли (preset ensembles):** Кнопки быстрой загрузки преднастроенных составов для текущего стиля. Каждый ансамбль — это набор инструментов с преднастроенными уровнями громкости, паттернами и voicing'ами. Пользователь выбирает ансамбль → все настройки инструментов применяются. После загрузки ансамбля можно тонко дотюнить любой инструмент вручную.

  Примеры ансамблей:

  | Ансамбль | Swing | Bossa Nova | Funk | Latin | Ballad |
  |---|---|---|---|---|---|
  | **Дуэт** | Piano + Upright Bass | Piano + Nylon Guitar | Piano + Electric Bass | Piano + Upright Bass | Piano + Upright Bass |
  | **Трио** | Piano + Upright Bass + Drums (jazz) | Piano + Upright Bass + Drums (light) | Piano + Electric Bass + Modern Kit | Piano + Upright Bass + Percussion Kit | Piano + Upright Bass + Drums (brushes) |
  | **Квартет** | Трио + Rhodes | Трио + Nylon Guitar | Трио + Electric Guitar | Трио + Trumpet | Трио + Rhodes (pads) |
  | **Квинтет** | Квартет + Trumpet (muted) | Квартет + Flute | Квартет + Rhodes (stabs) + Organ | Квартет + Flute + Percussion Kit | Квартет + Flute + Vibraphone |
  | **Полный ансамбль** | Все инструменты стиля включены | ← | ← | ← | ← |
  | **Сбросить** | Возвращает все настройки к стилевым дефолтам | ← | ← | ← | ← |

  **Зона 3 — Плитки инструментов (основная область страницы):**

  Каждый инструмент — отдельная **плитка-карточка**. Ничего не скрывается — все настройки инструмента видны в его плитке. Плитки идут подряд (grid 2–3 в ряд на десктопе, 1 в ряд на мобильном).

  ```
  ┌─ 🥁 Drums (Jazz Kit) ──────────────────────────────┐
  │  [●●● Вкл]  Громкость: [▬▬▬▬▬▬▬▬] 0.75             │
  │                                                       │
  │  Кит:      [ Jazz Kit ▾ ]                            │
  │  │  └─ доступно: Jazz Kit, Modern Kit                │
  │  Паттерн:  [ Ride-heavy swing ▾ ]                    │
  │                                                       │
  │  Звуки:                                               │
  │  Bass Drum  [●] [▬▬▬▬] 0.8    Snare  [●] [▬▬▬▬] 0.7 │
  │  Hi-Hat     [●] [▬▬▬▬] 0.6    Ride   [●] [▬▬▬▬] 0.9 │
  │  Crash      [●] [▬▬▬▬] 0.5    Rim    [○] [▬▬▬▬] 0.4 │
  │                                                       │
  │  Гуманизация: [ off | low | med | high ]             │
  │  Рандомайзер: [ off | subtle | moderate | high ]     │
  │  Fill частота: [ never | 4b | 8b | 16b ]             │
  └───────────────────────────────────────────────────────┘

  ┌─ 🎸 Bass (Upright Bass) ───────────────────────────┐
  │  [●●● Вкл]  Громкость: [▬▬▬▬▬▬▬▬] 0.8              │
  │                                                       │
  │  Тип баса: [ Upright Bass ▾ ]                        │
  │  │  └─ доступно: Upright Bass, Electric Bass         │
  │  Сложность: [ 1 | 2 | 3 | 4 | 5 | 6 | 7 ]           │
  │  Октава:    [ -1  |  0  | +1 ]                       │
  │                                                       │
  │  Рандомайзер: [ off | subtle | moderate | high ]     │
  └───────────────────────────────────────────────────────┘

  ┌─ 🎹 Piano (Grand Piano) ───────────────────────────┐
  │  [●●● Вкл]  Громкость: [▬▬▬▬▬▬▬▬] 0.85             │
  │                                                       │
  │  Профиль:  [ swing-sparse ▾ ]                        │
  │  Voicing:  [ shell2 | rootless3 | rootless4 | quartal ] │
  │                                                       │
  │  Рандомайзер: [ off | subtle | moderate | high ]     │
  └───────────────────────────────────────────────────────┘

  ┌─ 🎛️ Rhodes ────────────────────────────────────────┐
  │  [●●● Вкл]  Громкость: [▬▬▬▬▬▬▬▬] 0.6              │
  │                                                       │
  │  Режим слоя: [ subtle-offbeats ▾ ]                   │
  │  Voicing:    [ shell2 | rootless3 | rootless4 ]       │
  └───────────────────────────────────────────────────────┘
  ```

  **Ключевые принципы UX:**

  1. **Тогглы стилей** — не выпадающий список, а ряд кнопок. Все 5 стилей видны, выбор одним кликом. Удобно экспериментировать: включил swing → послушал → кликнул bossa → сравнил.
  2. **Плитка стиля** — темп и свинг всегда наверху. Кнопки ансамблей — быстрый старт для новичка или смена контекста для продвинутого.
  3. **Плитки инструментов** — каждый инструмент в своей карточке. Все настройки видны, ничего не скрыто. Вкл/Выкл — верхний переключатель: выключенный инструмент показывает плитку серой, но настройки сохраняются.
  4. **Ничего не скрывается** — никаких «развернуть/свернуть». Пользователь видит все настройки всех инструментов на одной странице (скролл). Это осознанное решение: музыканты привыкли к «приборной панели» (микшерный пульт, педалборд), где всё перед глазами.
  5. **Grid-раскладка** — на десктопе 2–3 плитки в ряд, на мобильном — 1 в ряд. Плитки адаптируются под ширину экрана.

  **Компактный вид на панели плеера:**

  ```
  [ Swing | Bossa Nova | Funk | Latin | Ballad ]  [ 140 BPM ⏎ ]  [ ⚙️ Инструменты ]
  ```

  - Стили — ряд кнопок-тогглов (компактные, без подписей — только иконки или короткие названия: Sw / Bo / Fu / La / Ba)
  - Темп — поле ввода с кнопками ±
  - ⚙️ Инструменты — открывает **модальное окно** с теми же плитками инструментов, что и на странице настроек, но в компактном виде (одна колонка, меньше отступов). В модальном окне — только инструменты, без стилей и ансамблей (стиль уже выбран на панели).

- **Конкуренты:** iReal Pro — плоский список настроек, нет группировки по стилям. Никто не даёт плиточный UX с ансамблями-предсетами.
- **Приоритет:** P0 (MVP) — ключевое для пользовательского опыта
- **Усиливает:** `core-player` (playbar + модальное окно инструментов), страница `/settings` (полные настройки), `music-core` (StyleProfile, preset ensembles)
- **Зависимости:** 3.1, 3.2

### 3.7. Интеллектуальная смена стиля: сохранять пользовательские правки

- **Проблема:** Если пользователь переключил стиль с Swing на Bossa, а потом обратно — его кастомные настройки для Swing потеряются. Нужна «память» пользовательских правок.
- **Описание:** Система хранит два слоя настроек:
  - **Базовый слой** — `StyleProfile.instrumentDefaults` (дефолты стиля, неизменны)
  - **Пользовательский слой** — `UserSettings.perStyleOverrides[style]` (только то, что пользователь изменил)

  Алгоритм: при выборе стиля применяются базовые дефолты, затем поверх накладываются пользовательские overrides. При переключении туда-обратно пользовательские правки сохраняются.

  Кнопка «Сбросить на стиль» очищает пользовательский слой для текущего стиля.
- **Приоритет:** P1
- **Усиливает:** `shared` (UserSettingsDTO), `music-core` (StyleProfile), `apps/api` (settings service)
- **Зависимости:** 3.1

## 4. Улучшения существующих функций

### 4.1. Рефакторинг `Style`: от enum к объекту первого класса

- Текущий `type Style = 'swing' | 'bossa' | 'funk' | 'latin' | 'ballad'` остаётся как идентификатор

- Миграция обратно-совместимая: `Style` продолжает работать, `StyleProfile` — новый API

### 4.2. Миграция `drumsPattern` → полное упразднение

- Новые настройки Drums: `drumKit` (выбор манифеста: `jazz-kit` | `modern-kit`)

### 4.3. Расширение `InstrumentManifest` стилевыми дефолтами

  ```ts
  interface InstrumentManifest {
    // ... существующие поля
    /** Дефолтные настройки для каждого стиля (переопределяют defaultSettings) */
    perStyleDefaults?: Record<Style, Record<string, unknown>>;
  }
  ```

### 4.4. Piano: новые montuno-паттерны для Latin

- Добавить «кирпичик» `montuno` в `pianoComping.ts` — характерный латиноамериканский ритмический рисунок

### 4.5. Rhodes: высокий регистр для Latin

## 6. Риски и допущения

| Риск | Вероятность | Решение |
|---|---|---|
| Новые сэмплы (8 инструментов) требуют значительного времени на поиск/запись/редактирование | Высокая | Приоритизировать по стилям: сначала Modern Kit + Electric Bass (для MVP funk-стиля), затем Nylon Guitar + Percussion Kit, остальные — во вторую очередь. Использовать CC0/свободные библиотеки |
| Увеличение числа инструментов до 15+ замедляет загрузку | Средняя | Lazy loading: сэмплы загружаются только для enabled инструментов в текущем стиле. Неиспользуемые манифесты не загружаются |
| Трёхуровневый UI настроек может быть сложен в реализации | Средняя | Начать с MVP: стиль → список инструментов с Вкл/Громкость. Детальные настройки — позже через раскрытие ⚙️ |
| Per-style overrides усложняют модель данных настроек | Средняя | Хранить как JSON-поле `perStyleOverrides` в `user_settings`. Валидировать через Zod-схему |
| Новый `StyleProfile` может конфликтовать с существующим `InstrumentManifest` | Низкая | `StyleProfile` — новый тип, не трогает `InstrumentManifest`. Инструменты получают стилевые настройки через `setStyleProfile()`, не через конструктор |
| Пользовательские настройки могут «поехать» при миграции | Низкая | Обратная совместимость: старые поля не удаляются, `perStyleOverrides` заполняется только при ручном изменении пользователем |
| Сэмплы для трубы, флейты и кларнета требуют реалистичного legato | Высокая | [assumption] Первая версия — без legato, монофонические ноты. Legato-переходы — в следующей версии |
| Стиле-специфичные паттерны для гитары (bossa-comping) требуют нового алгоритма | Средняя | [assumption] Реализовать как отдельный планировщик в `GuitarInstrument.schedule()`, аналогично bass-стилям |

## 7. Метрики успеха

- **Полнота стилей:** Каждый из 5 стилей имеет полный ростер (required + recommended) с корректными дефолтными настройками — пользователь выбирает стиль и получает аутентичное звучание без ручной настройки
- **Стилевой свинг:** Каждый стиль имеет характерный swing ratio, который применяется ко всем инструментам согласованно
- **Стилевой темп:** При выборе стиля темп автоматически выставляется в стилевой дефолт (с возможностью ручной корректировки)
- **UX настроек:** Пользователь видит только релевантные для стиля инструменты; нерелевантные скрыты (но доступны через «Добавить инструмент»)
- **Сохранение правок:** При переключении стиля туда-обратно пользовательские настройки сохраняются
- **Новые инструменты:** Modern Kit и Electric Bass загружаются и играют без задержек; Nylon Guitar даёт характерный bossa-звук
- **Производительность:** 15+ инструментов в реестре не замедляют загрузку — сэмплы загружаются on-demand
- `typecheck` + `lint` + `test` — зелёные
- Все существующие тесты проходят (с учётом миграции)
- Документация отражает актуальное поведение
