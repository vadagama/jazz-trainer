# VISION — Рефакторинг гармонии: Piano, Rhodes и глобальные стили

**Дата:** 2026-06-13
**Горизонт:** 1 версия (3–4 недели)
**Статус:** 🟢 Принято

## 1. Резюме (Executive Summary)

Рефакторинг гармонической подсистемы Jazz Trainer с тремя целями: (1) поднять выбор стиля на глобальный уровень — единый переключатель swing/bossa/funk/latin/ballad на панели плеера, влияющий на все инструменты сразу; (2) создать новый инструмент **Piano** на базе сэмплов Salamander Grand Piano и Upright KW — он берёт на себя роль основного гармонического инструмента с богатыми компинг-паттернами и voicings; (3) переопределить роль **Rhodes** как дополнительного слоя для создания плотных, студийных аранжировок (pads, разреженные паттерны, ambient-текстуры).

Ключевая ценность: пользователь получает «живой» аккомпанемент, приближенный к реальному джазовому ансамблю — барабаны, бас, фортепиано и Rhodes работают в одном стиле и дополняют друг друга, а не конфликтуют. Переключение стиля одним кликом меняет поведение всей ритм-секции.

## 2. Конкурентный контекст

| Конкурент                 | Гармонический инструмент                             | Глобальный стиль                                      | Дополнительные слои                    |
| ------------------------- | ---------------------------------------------------- | ----------------------------------------------------- | -------------------------------------- |
| **iReal Pro**             | MIDI-фортепиано (band-in-a-box)                      | Стиль выбирается глобально, влияет на все инструменты | Нет — один фортепианный трек           |
| **EarMaster**             | Нет аккомпанемента                                   | Н/Д                                                   | Н/Д                                    |
| **Jazz Trainer (сейчас)** | Только Rhodes (13 ритм. режимов + 3 voicing density) | Стиль только в Drums (`drumsPattern`)                 | Нет слоёв                              |
| **Jazz Trainer (цель)**   | Piano (основной) + Rhodes (доп. слой)                | Глобальный стиль влияет на все 4 инструмента          | Piano + Rhodes = управляемая плотность |

**Ниша после рефакторинга:** Jazz Trainer — единственный веб-тренажёр с многослойной гармонической подсистемой (piano + rhodes layer), настоящими стерео-сэмплами акустического фортепиано и глобальным стилем, синхронизирующим всё. iReal Pro делает band-in-a-box на MIDI-звуках без слоёв. Никто не даёт управляемую плотность аранжировки (solo piano → piano + rhodes).

## 3. Новые функции (Features)

### 3.1. Глобальный стиль (Style Selector)

- **Проблема:** Сейчас выбор стиля (`swing`/`bossa`/`funk`) зашит в настройки барабанов (`drumsPattern`). Смена стиля не влияет на Rhodes и Bass — они продолжают играть в своём режиме. Пользователь не может одним действием переключить весь ансамбль.
- **Описание:** Стиль становится глобальной настройкой верхнего уровня (как темп). Отображается на панели плеера в виде селектора: `Swing` / `Bossa Nova` / `Funk` / `Latin` / `Ballad`. При смене стиля:
  - **Drums** — переключает внутренний паттерн (уже умеет, нужно только переподключить с `drumsPattern` на глобальный `style`)
  - **Piano** — адаптирует компинг-паттерн (в свинге активнее, в балладе разреженнее)
  - **Rhodes** — адаптирует комплементарный слой (в bossa — ambient pads, в funk — stabs, в ballad — мягкие целые)
  - **Bass** — адаптирует басовую линию (walking bass в свинге, two-feel в балладе, latin-паттерн в bossa)
- **Конкуренты:** iReal Pro — глобальный стиль есть, но влияет только на MIDI-паттерны; EarMaster — нет.
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `core-player` (панель плеера), `music-core` (StyleManager/адаптация)
- **Зависимости:** 3.2 (Piano), 3.3 (Rhodes). Требует депрекации `drumsPattern` → `style`.

### 3.2. Piano Instrument (новый инструмент)

- **Проблема:** Сейчас единственный гармонический инструмент — Rhodes. Его характерный «звенящий» тембр хорош, но пользователь не может практиковаться под звук акустического фортепиано — основного инструмента джазовой практики. Паттерны Rhodes (13 ритмических режимов) ограничены одним инструментом.
- **Описание:** Новый `PianoInstrument` в `music-core`:
  - **Сэмплы:** Salamander Grand Piano (44.1kHz, 16bit, стерео) — основной набор. Upright KW (Best quality) — альтернативный. Пользователь переключает библиотеку в настройках → подгружаются нужные сэмплы.
  - **Оптимизация сэмплов:** 2–4 velocity layers (soft, medium, hard, [bark]), урезанный диапазон нот (~C3–C6 — без крайнего баса и высоких), конвертация WAV → OGG (libopus ~128k). Итоговый размер набора: ~15–25 MB.
  - **Компинг-паттерны (составные профили):** Вместо одиночных ритмических режимов (как в текущем Rhodes), Piano использует **4-тактовые составные профили** — последовательность из простых паттернов, меняющихся каждый такт. Это создаёт живое, развивающееся звучание без повторяющейся «драм-машины». Базовые «кирпичики» — текущие режимы Rhodes: `charleston`, `reverse-charleston`, `basie-2-4`, `offbeat-2-4`, `anticipation-4and`, `one-twoand-four`, `oneand-three`, `twoand-only`, `four-and-sparse`, `two-threeand`, `halfNotes`, `quarterNotes`, `wholeNotes`, `rest`.

    **Профили компинга (5 шт. для MVP):**
    1. **Swing Sparse** (сложность: 1/3)
       - Такт 1: Charleston
       - Такт 2: Rest
       - Такт 3: 2& only
       - Такт 4: 4& anticipation

    2. **Swing Medium** (сложность: 2/3)
       - Такт 1: Charleston
       - Такт 2: Reverse Charleston
       - Такт 3: 1 + 2& + 4
       - Такт 4: 4& anticipation

    3. **Basie Light** (сложность: 2/3)
       - Такт 1: 2 and 4
       - Такт 2: Charleston
       - Такт 3: 2& only
       - Такт 4: 2 and 4

    4. **Offbeat Push** (сложность: 3/3)
       - Такт 1: 2& only
       - Такт 2: Offbeat 2& 4&
       - Такт 3: Charleston
       - Такт 4: 4& anticipation

    5. **Beginner Safe** (сложность: 1/3)
       - Такт 1: beat 1 only
       - Такт 2: beat 1 + 3
       - Такт 3: Charleston
       - Такт 4: Rest

    **Модель данных профиля:**

    ```ts
    interface CompingProfile {
      id: string;
      name: string;
      complexity: 1 | 2 | 3;
      /** 4-bar cycle of simple patterns */
      bars: [CompPatternId, CompPatternId, CompPatternId, CompPatternId];
    }
    ```

    Профили — это данные (не код), легко расширяются. Пользователь выбирает профиль в настройках (`pianoProfile`). В будущем — редактор профилей.

  - **Voicings:** `shell2`, `rootless3`, `rootless4` + `quartal` (открытые квартовые созвучия)
  - **FX-цепь:** Convolution Reverb (IR зала) + subtle EQ. Без хоруса и тремоло (в отличие от Rhodes).
  - **Человечность:** Timing jitter + velocity variation (как в Rhodes сейчас).

- **Конкуренты:** iReal Pro — MIDI-фортепиано (звук низкого качества); JazzPianoSchool — pre-recorded треки, нет живого компинга; **ни один конкурент не даёт настоящее сэмплированное фортепиано с управляемым компингом в реальном времени.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `music-core` (PianoInstrument, pianoVoicing, pianoManifest)
- **Зависимости:** 3.1 (глобальный стиль). Сэмплы Salamander Grand Piano + Upright KW.

### 3.3. Rhodes как комплементарный слой

- **Проблема:** Сейчас Rhodes — единственный гармонический инструмент. При добавлении Piano нужно переопределить его роль так, чтобы он не конфликтовал, а дополнял.
- **Описание:** Rhodes получает новый набор «комплементарных» режимов, специально спроектированных для работы поверх Piano:
  - `pads` — целые ноты with slow attack (Tone.js amplitude envelope), создаёт гармоническую подушку
  - `subtle-offbeats` — тихие аккорды только на «и» (2&, 4&), добавляет ритмическую текстуру без конфликта с piano
  - `high-comping` — компинг в верхнем регистре (выше C4), не пересекается с piano в среднем
  - `ambient-swells` — долгие аккорды с медленной атакой, срабатывают раз в 2–4 такта
  - `stab-accents` — короткие аккордовые акценты на сильные доли (для funk)
  - `none` — Rhodes выключен (solo piano)
  - **Настройка плотности:** `rhodesLayerMode` — выбор комплементарного режима + `rhodesLayerVolume` (относительная громкость слоя).
  - **Взаимодействие с Piano:** Правила избегания конфликтов — Rhodes не играет в той же ритмической позиции, что и piano (задержка на 1/16 или сдвиг по долям); предпочитает верхний регистр когда piano в среднем.
  - **Voice leading:** Rhodes использует другой набор voicings (более открытые, quartal, кластеры), чтобы не дублировать piano.
- **Конкуренты:** iReal Pro — нет слоёв; **никто не предлагает управляемую плотность гармонической аранжировки.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `music-core` (RhodesInstrument — новые complementary modes)
- **Зависимости:** 3.2 (Piano). Требует рефакторинга `RhodesInstrument` — отделение комплементарных паттернов от основных.

### 3.4. Панель плеера: Style Selector

- **Проблема:** Выбор стиля спрятан в Settings (отдельная страница). Пользователь не может быстро переключить стиль во время игры.
- **Описание:** На панели плеера (рядом с транспортом) появляется **Style Selector** — выпадающий список / кнопки: `Swing` / `Bossa Nova` / `Funk` / `Latin` / `Ballad`. Мгновенное переключение без остановки воспроизведения. Все остальные настройки (Harmony Density, громкость инструментов, выбор профиля) остаются в глобальных Settings.
- **Конкуренты:** iReal Pro — глобальный стиль на основном экране; EarMaster — нет.
- **Приоритет:** P1
- **Усиливает плагин / Новая категория:** `core-player` (playbar)
- **Зависимости:** 3.1, 3.2, 3.3.

### 3.5. Piano Randomizer (аналог DrumRandomizer)

- **Проблема:** Фиксированный компинг-паттерн звучит как драм-машина — нужна управляемая вариативность.
- **Описание:** `PianoRandomizer` — слой над базовыми компинг-паттернами. На каждом такте:
  - Варьирует ритмический паттерн (пропускает долю, добавляет anticipation, смещает на восьмую)
  - Варьирует voicing density (shell → rootless4) для динамического разнообразия
  - Добавляет passing chords между аккордами прогрессии
  - Уровни: `off` / `subtle` / `moderate` / `high`
- **Конкуренты:** Никто не предлагает управляемую рандомизацию фортепианного компинга.
- **Приоритет:** P2
- **Усиливает плагин / Новая категория:** `music-core` (PianoRandomizer)
- **Зависимости:** 3.2.

## 4. Улучшения существующих функций

### 4.1. Миграция `drumsPattern` → `style`

- Поле `drumsPattern` в `UserSettingsDTO` и `DrumInstrumentSettings` депрекейтится
- Добавляется новое поле `style` на верхнем уровне `UserSettingsDTO`
- При чтении старых настроек: `drumsPattern` мапится в `style` (если `style` не задан)
- `DrumInstrument` читает `style` из глобального контекста, а не из своих настроек
- Старые поля сохраняются на переходный период с пометкой `@deprecated`

### 4.2. Миграция Rhodes-настроек

- Текущие поля `rhodesMode`, `rhodesVoicingDensity` остаются — но теперь управляют **комплементарным** слоем
- Добавляется `rhodesLayerMode` — выбор комплементарного паттерна (pads, subtle-offbeats, high-comping, ambient-swells, stab-accents, none)
- Простые компинг-режимы («charleston», «basie-2-4» и т.д.) становятся «кирпичиками» для Piano-профилей и удаляются из Rhodes-настроек

### 4.3. Новые Piano-настройки

- `pianoEnabled` (boolean) — вкл/выкл Piano
- `pianoVolume` (0–1) — громкость
- `pianoProfile` (string) — выбор профиля компинга: `swing-sparse`, `swing-medium`, `basie-light`, `offbeat-push`, `beginner-safe`
- `pianoVoicingDensity` — `shell2` | `rootless3` | `rootless4` | `quartal`
- `pianoSampleLibrary` — `salamander` | `upright-kw`
- `pianoRandomizationLevel` — `off` | `subtle` | `moderate` | `high` (после 3.5)

### 4.4. Обновление `docs/RHODES.md` и создание `docs/PIANO.md`

- `docs/RHODES.md` — обновляется: новая роль комплементарного слоя
- `docs/PIANO.md` — новый документ: сэмплы, паттерны, voicings, взаимодействие с Rhodes

### 4.5. Стиле-зависимый Bass

- `BassInstrument` получает адаптацию к глобальному стилю:
  - Swing → walking bass (как сейчас)
  - Bossa → latin bass pattern (root-5th)
  - Funk → syncopated bass line
  - Ballad → two-feel, разреженный
  - Latin → montuno-style bass

## 5. Не войдёт в эту версию (Out of Scope)

- MIDI-вывод для Piano/Rhodes (Фаза 5, 🔴)
- Визуализация фортепианной клавиатуры
- Пользовательские компинг-паттерны (редактор)
- Piano-заполнения (fills) такой же сложности как DrumRandomizer fills
- Более 2 библиотек сэмплов (Salamander + Upright KW — достаточно)
- Автоматическое определение стиля из DSL-разметки сетки
- «Интеллектуальный» выбор voicings под конкретный аккорд (пока что voicings выбираются по density)
- Ритмическая синхронизация bass drum ↔ bass line, snare ↔ piano акценты

## 6. Риски и допущения

| Риск                                                                | Решение                                                                                                                   |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| Salamander Grand Piano слишком тяжёлый для веба                     | Урезаем до 2–4 velocity layers, ~C3–C6, OGG 128kbps → 15–25 MB. Используем Tone.js `Sampler` с lazy loading.              |
| Два фортепианных тембра одновременно не нужны                       | Сэмплы загружаются on-demand: при выборе библиотеки в настройках старая выгружается, новая подгружается                   |
| Rhodes и Piano могут конфликтовать по частотам/ритму                | Чёткие правила: комплементарные паттерны избегают долей, где активен piano; разные регистры                               |
| Миграция `drumsPattern` → `style` ломает пользовательские настройки | Обратная совместимость: `drumsPattern` мапится в `style` при чтении. Старое поле остаётся deprecated на переходный период |
| Увеличение числа настроек пугает пользователя                       | Панель плеера показывает только ключевые контролы (style, density, громкость). Полные настройки — в Settings              |
| Конвертация .sf2/.sfz сэмплов может потребовать MIDI-маппинга       | Берём уже готовый набор из freepats.zenvoid.org (44.1kHz 16bit) и конвертируем ffmpeg                                     |

## 7. Метрики успеха

- Пользователь переключает стиль на панели плеера → все 4 инструмента меняют поведение в течение одного такта
- Piano играет полноценный компинг с выбранным ритмическим режимом и voicing density
- Rhodes можно включить как слой → звучание становится плотнее без артефактов и ритмических конфликтов
- Переключение библиотеки сэмплов (Salamander ↔ Upright KW) работает без перезагрузки страницы
- `typecheck` + `lint` + `test` — зелёные
- Все существующие тесты проходят (с учётом миграции полей)
- Документация (`PIANO.md`, обновлённый `RHODES.md`) отражает актуальное поведение
