# VISION — Рефакторинг DrumInstrument

**Дата:** 2026-06-13
**Горизонт:** 3 этапа (Swing → Bossa Nova → Funk)
**Статус:** 🟢 Принято

## 1. Резюме (Executive Summary)

Рефакторинг инструмента барабанов с заменой текущей минимальной реализации (3 звука: ride, stir, hihatFoot) на полноценную ритм-секцию на базе бесплатной библиотеки сэмплов [Swirly Drums](https://shop.karoryfer.com/pages/free-swirly-drums) (4700+ сэмплов, brushes).

Три этапа, каждый добавляет новый стиль: **классический свинг** (ride, hihat, snare, bass drum, crash, rim), **босса-нова** (cross-stick rim, clave-паттерн), **фанк** (groove-ориентированные паттерны с синкопами). Барабаны работают в общем `TransportEngine` с басом и Rhodes, разделяя `swingRatio` и не конфликтуя по миксу. На следующих итерациях — ритмическая синхронизация бас-бочки с басовой линией и snare с аккордовыми акцентами.

## 2. Конкурентный контекст

| Конкурент | Ритм-секция | Стили | Сэмплы |
|---|---|---|---|
| **iReal Pro** | Band-in-a-box: drums, bass, piano | Swing, Latin, Funk, Pop | Встроенные MIDI-звуки |
| **EarMaster** | Метроном + ритм-упражнения | Только метроном | Нет |
| **Jazz Trainer (сейчас)** | 3 звука, 1 паттерн | Только swing 4/4 | 12 OGG-файлов |

**Ниша Jazz Trainer после рефакторинга:** единственный веб-тренажёр с настоящими brushes-сэмплами и тремя джазовыми стилями в одной ритм-секции.

## 3. Новые функции (Features)

### 3.1. Звуковой набор Swirly Drums

- **Проблема:** Текущие 3 звука (ride, stir, hihatFoot) не образуют полноценную ритм-секцию — нет бас-бочки и малого барабана.
- **Описание:** Пользователь слышит 6 типов звуков из библиотеки Swirly Drums, каждый с round-robin сэмплами (4 варианта) и humanization.
- **Конкуренты:** iReal Pro имеет полный набор, но на MIDI-звуках низкого качества.
- **Приоритет:** P0 (MVP Этапа 1)
- **Усиливает плагин / Новая категория:** `music-core` (DrumInstrument)
- **Зависимости:** Загрузка и конвертация сэмплов Swirly Drums в OGG.

### 3.2. Паттерн «Классический свинг»

- **Проблема:** Текущий `swingRide` — это только ride с оффбитами. Нет snare backbeat, нет bass drum feathering, нет crash-акцентов.
- **Описание:** Полноценный джазовый свинг-паттерн: ride (ding ding-a-ding), hihat восьмые (variable openness), snare на 2 и 4, bass drum feathering (тихие четверти, акцент на 1 и 3), crash на начало формы.
- **Конкуренты:** iReal Pro — есть; EarMaster — нет; Tenuto — нет.
- **Приоритет:** P0 (MVP Этапа 1)
- **Усиливает плагин / Новая категория:** `music-core` (DrumInstrument)
- **Зависимости:** 3.1 (звуковой набор), общий `swingRatio` из `TransportEngine`.

### 3.3. Паттерн «Босса-нова»

- **Проблема:** Нет поддержки латинских стилей — пользователь не может практиковать босса-нову (Girl from Ipanema, Blue Bossa и др.).
- **Описание:** Clave-подобный паттерн на rim (cross-stick), bass drum синкопы (1 и 3&), hihat chick на 2 и 4, восьмые на остальных. Straight feel (swingRatio ~0.50).
- **Конкуренты:** iReal Pro — есть (Latin стиль); EarMaster — нет.
- **Приоритет:** P1 (Этап 2)
- **Усиливает плагин / Новая категория:** `music-core` (DrumInstrument)
- **Зависимости:** 3.1 (звуковой набор), `pattern` selector в настройках.

### 3.4. Паттерн «Фанк»

- **Проблема:** Нет поддержки groove-ориентированных стилей для современных джазовых прогрессий.
- **Описание:** Агрессивный грув: 16th note hihat, синкопированная bass drum, snare акценты на 2 и 4, опциональные заполнения. Straight feel с возможностью 16th-note swing.
- **Конкуренты:** iReal Pro — есть (Funk стиль); остальные — нет.
- **Приоритет:** P2 (Этап 3)
- **Усиливает плагин / Новая категория:** `music-core` (DrumInstrument)
- **Зависимости:** 3.1 (звуковой набор), `pattern` selector, `funkComplexity`.

### 3.5. Настройки барабанов (пользовательские)

- **Проблема:** Текущие настройки покрывают только 3 звука и 1 паттерн.
- **Описание:** Полный набор контролов: выбор стиля (swing/bossa/funk), per-sound enable/volume для 6 звуков, hihat openness (0–5), swing feel (0.50–0.75), humanize intensity (off/low/med/high).
- **Конкуренты:** iReal Pro — миксер инструментов (volume, reverb), но без per-sound контролов и humanization.
- **Приоритет:** P0 (начинается на Этапе 1, расширяется на Этапах 2–3)
- **Усиливает плагин / Новая категория:** `core-player` (SettingsForm), `api` (user_settings)
- **Зависимости:** 3.1, 3.2.

### 3.6. Ритмическая синхронизация с басом и Rhodes (будущее)

- **Проблема:** Барабаны, бас и Rhodes работают независимо — bass drum не подчёркивает басовую линию, snare не акцентирует аккордовые смены.
- **Описание:** `TransportEngine` получает «groove template» — общий ритмический слой, которому следуют все инструменты. Bass drum акцентирует сильные доли баса, snare подчёркивает аккордовые смены Rhodes.
- **Конкуренты:** iReal Pro — частично (стиль определяет паттерны всех инструментов); остальные — нет.
- **Приоритет:** P3 (после Этапа 3)
- **Усиливает плагин / Новая категория:** `music-core` (TransportEngine), все инструменты
- **Зависимости:** 3.2, 3.3, 3.4. Требует архитектурного решения (ADR).

### 3.7. Рандомизация барабанных паттернов (Variation Engine)

- **Проблема:** Человеческий джазовый барабанщик никогда не играет два chorus'а одинаково — он варьирует ride-паттерн (ding ding-a-ding → разреженный → busy), добавляет ghost notes на snare между backbeat, меняет плотность bass drum feathering, играет заполнения на границах формы. Текущие фиксированные паттерны + micro-humanization (timing/velocity jitter) звучат как драм-машина, а не как живой музыкант.
- **Описание:** `DrumRandomizer` — слой над базовыми паттернами всех трёх стилей. На каждом такте принимает пер-тактные и пер-долевые решения по заданным вероятностным правилам:
  - **Уровни рандомизации** (`randomizationLevel`): `off` (базовый паттерн + micro-humanization), `subtle` (~10% тактов с вариацией, ghost notes редко), `moderate` (~25% тактов, регулярные ghost notes, fill каждые 8 тактов), `high` (активная вариативность, fill каждые 4 такта).
  - **Per-element toggles:** `rideVariation`, `snareGhosts`, `bassDrumVariation` — позволяют отключить вариативность конкретного звука. Например: оставить ride стабильным как ориентацию для practising, но варьировать snare для organic feel.
  - **Заполнения (fills):** `fillFrequency` (`never` / `4bars` / `8bars` / `16bars`) и `fillComplexity` (`simple` / `medium` / `complex`). Fill адаптируется под стиль: в свинге — snare + bass drum комбинации с triplet-feel, в боссе — rim-вариации clave, в фанке — busy fills с crash-акцентами на 16-х.
  - **Музыкальные ограничения:** ghost notes только на слабых долях (e, a), ride не варьируется на последнем такте формы (чтобы не сбить), fill не перекрывает мелодическую фразу.
  - **Архитектура:** `DrumRandomizer` — чистый класс в `music-core`, без IO, полностью тестируемый. `DrumInstrument.schedule()` вызывает `randomizer.apply(basePattern, settings, barContext)` → получает модифицированный набор `DrumEvent[]` и планирует его. При `randomizationLevel = 'off'` — zero-cost: базовый паттерн проходит без изменений.
- **Конкуренты:** iReal Pro — фиксированные паттерны без controlled randomization; EarMaster — нет; **ни один конкурент не предлагает управляемую рандомизацию джазовых барабанов** с per-element контролем.
- **Приоритет:** P1 (Этап 2 — одновременно с босса-новой, применяется ко всем трём стилям)
- **Усиливает плагин / Новая категория:** `music-core` (DrumRandomizer + DrumInstrument)
- **Зависимости:** 3.2 (swing-паттерн — первый получает рандомизацию), 3.3 (bossa-паттерн), 3.4 (funk-паттерн), 3.5 (настройки — новые поля в SettingsForm).

## 4. Улучшения существующих функций

### 4.1. Миграция настроек пользователей

Текущие поля `drumsStir*`, `drumsHihat*`, `drumsRide*` мапятся на новые (`drumsSnare*`, `drumsHihat*`, `drumsRide*`). Добавляется миграция БД. Старые поля деприкейтятся, но сохраняются для обратной совместимости на время переходного периода.

### 4.2. docs/DRUMS.md

Документация обновляется: новые звуки, новые паттерны, описание библиотеки Swirly Drums, инструкция по добавлению стилей.

## 5. Не войдёт в эту версию (Out of Scope)

- Сэмплы, записанные палочками (отдельная библиотека)
- Генератор пользовательских ритмических паттернов
- MIDI-вывод барабанов (Фаза 5, 🔴)
- Визуализация ударной установки
- Аудио-эффекты на мастер-канале (compressor, reverb)
- Ритмическая синхронизация с басом и Rhodes (Этап 2+: P3)
- `seed` для повторяемой рандомизации (пользователь не запросил)
- `dynamicRange` — динамическая арка по секциям формы (→ вместе с 3.6)
- Кастомные правила рандомизации на пользователя (слишком сложно для V1)

## 6. Риски и допущения

| Риск | Решение |
|---|---|
| 4700 сэмплов — слишком много для веб-загрузки | Выбираем подмножество (~60–100 файлов): 1 velocity layer на звук, 4 RR. Остальное — lazy load |
| Размер страницы с полным набором | Базовый свинг-кит (~30 файлов), босса/фанк подгружаются on-demand |
| Swirly Drums — brushes, не палочки | Идеально для джаза/босса-новы. Фанк потребует эквалайзера в цепочке эффектов (на Этапе 3) |
| Миграция настроек пользователей | Старые поля деприкейтим, мапим на новые. Добавляем миграцию |
| Конвертация WAV → OGG | Используем ffmpeg/libopus 128k — как уже делали для текущих сэмплов |

## 7. Метрики успеха

- Пользователь может выбрать стиль (swing/bossa/funk) и слышит соответствующий паттерн
- Все 6 звуков имеют независимый on/off и volume
- Humanization создаёт естественное звучание (нет «machine gun effect»)
- Переключение стиля не ломает воспроизведение
- Рандомизация создаёт заметную, но музыкальную вариативность (нет хаоса, нет «machine gun effect»)
- Per-element toggles работают: отключение `rideVariation` фиксирует ride-паттерн
- Переключение `randomizationLevel` с `off` на `high` мгновенно меняет поведение
- Все существующие тесты проходят, новые тесты покрывают 3 стиля + рандомизацию
- `typecheck` + `lint` + `test` — зелёные
