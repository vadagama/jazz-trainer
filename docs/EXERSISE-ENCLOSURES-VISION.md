## 1. Резюме (Executive Summary)

**Опевания** (enclosures / approach notes) — третий тип упражнений в плагине `practice-cards`, изначально запланированный в `EXERSISE-VISION.md` §5 как «скоро». Плитка «Опевания» уже есть в `StepTypeSelect`, но сейчас `disabled: true`. Задача этой итерации — активировать плитку и реализовать полноценное упражнение.

По замыслу пользователя **упражнения по опеваниям делаются по образцу гамм**:

- Тот же discriminated union `ExerciseConfig`.
- Та же базовая конфигурация `BaseExerciseConfig` (тональности, повторы, карточки, аккомпанемент, метроном).
- Те же источники гармонии (`ChordSource`): `unified` (отдельно), `pattern`, `random`, `dsl`.
- Те же хелперы генерации из `core.ts` (`extractChordsFromSource`, `toPracticeBars`, `repeatBars`, `expandBarsPerChord`, `shuffle`).
- Те же режимы отображения карточек (`current` / `prev-current` / `prev-current-next`).

Отличие от гамм — вместо лада на карточке показывается **целевая ступень аккорда и ноты оборота**, а пользователь тренирует мелодический оборот вокруг аккордового звука.

**Ключевая ценность:** пользователь видит аккорд + целевую ступень + ноты опевания, слышит живой аккомпанемент и отрабатывает джазовый мелодический словарь. Это мост между «знаю аккордовые звуки» и «умею строить мелодию» — в формате потока карточек, который не предлагает ни один конкурент.

**MVP:** 6 базовых типов опеваний, целевые ступени `1, 3, 5, 7`, два режима — «отдельно» (`unified`) и «по прогрессии» (`pattern` / `random` / `dsl`).

## 2. Что такое опевание (музыкальная справка)

**Опевание (enclosure)** — мелодический приём, при котором целевой аккордовый звук окружается соседними нотами сверху и/или снизу, создавая тяготение и джазовую окраску. Фундамент бибопа.

### 2.1. Базовые типы опеваний (MVP)

| Тип | Формула | Пример для цели E (в C) | Название в UI |
| --- | --- | --- | --- |
| **Diatonic upper** | ↑2 → цель | `F → E` | Диатонический сверху |
| **Diatonic lower** | ↓2 → цель | `D → E` | Диатонический снизу |
| **Chromatic upper** | ↑½ → цель | `F → E` | Хроматический сверху |
| **Chromatic lower** | ↓½ → цель | `E♭ → E` | Хроматический снизу |
| **Full diatonic enclosure** | ↑2 → ↓2 → цель | `F → D → E` | Полное диатоническое |
| **Full chromatic enclosure** | ↑½ → ↓½ → цель | `F → E♭ → E` | Полное хроматическое |

> В UI добавляется опция **«Все (случайно)»** — на каждый такт случайно выбирается один из 6 типов.

### 2.2. Целевые ступени

В MVP поддерживаются только аккордовые звуки:

| Ступень | Важность |
| --- | --- |
| **3** | Критичная — определяет мажор/минор |
| **7** | Критичная — определяет функцию аккорда |
| **5** | Стабильный звук |
| **1** | Тоника |

Пользователь может выбрать одну или несколько ступеней. Если выбрано несколько — в режиме `over-chords` на каждый аккорд выпадает одна случайная ступень из выбранных. Если выбрана одна — отработка только этой ступени.

### 2.3. Почему опевания похожи на гаммы

| Аспект | Гаммы | Опевания |
| --- | --- | --- |
| Режим «отдельно» | `source.type === 'unified'` | `source.type === 'unified'` |
| Режим «по прогрессии» | `source.type === 'pattern' / 'random' / 'dsl'` | `source.type === 'pattern' / 'random' / 'dsl'` |
| Карточка | аккорд + лад | аккорд + ступень + ноты оборота |
| Что играет пользователь | лад вверх/вниз | оборот вокруг ступени |
| Генератор | `scaleExercise.ts` | `enclosureExercise.ts` |
| UI-шаг | `StepScaleConfig.tsx` | `StepEnclosureConfig.tsx` |

## 3. Новые функции (Features)

### 3.1. Активация плитки «Опевания»

- **Проблема:** плитка «Опевания» существует, но `disabled: true`, а тип `onSelect` не включает `'embellishments'`.
- **Описание:** снять `disabled` с плитки, заменить иконку `⏳` → `🎵`, расширить типы `onSelect` и `ExerciseKind` до `'chords' | 'scales' | 'embellishments'`.
- **Приоритет:** P0
- **Зависимости:** —

### 3.2. Мастер настройки опеваний

- **Проблема:** пользователь должен выбрать тип опевания, целевые ступени и источник гармонии.
- **Описание:** новый компонент `StepEnclosureConfig.tsx` (по образцу `StepScaleConfig.tsx`):
  - **Тип опевания:** выпадающий список из 6 типов + «Все (случайно)».
  - **Целевые ступени:** чекбоксы `1, 3, 5, 7` (множественный выбор).
  - **Октавы:** `1` или `2`.
  - **Источник:** `SourceCard` с теми же вкладками, что и у гамм (`unified`, `pattern`, `random`, `dsl`).
  - Остальные параметры — общие карточки: тональности, повторы, бесконечность, затакт, режим карточек, аккомпанемент, метроном, темп, рандомизация, тактов на элемент.
- **Приоритет:** P0
- **Зависимости:** 3.1

### 3.3. Генератор упражнений по опеваниям

- **Проблема:** нужен чистый модуль `Config → PracticeBar[]`, аналогичный `scaleExercise.ts`.
- **Описание:** реализовать `generateEnclosureExercise(config, rng)`:
  - При `source.type === 'unified'` — строить тонический аккорд для каждой выбранной тональности и генерировать обороты для каждой выбранной ступени.
  - При `source.type !== 'unified'` — получить прогрессию через `extractChordsFromSource`, на каждый аккорд назначить одну случайную ступень из выбранных.
  - Поддержать `playRandomly`, `barsPerChord`, `repetitions`, `infinite`, `octaves` — теми же утилитами, что и гаммы.
- **Приоритет:** P0
- **Зависимости:** 3.4 (доменный модуль `enclosures.ts`)

### 3.4. Доменный модуль опеваний в `music-core`

- **Проблема:** расчёт нот оборота — чистая музыкальная теория, которая пригодится и для будущей MIDI-оценки, и для лекции `theory.approach-notes`.
- **Описание:** создать `packages/music-core/src/chords/enclosures.ts`:
  - Типы: `EnclosureType`, `ConcreteEnclosureType`, `TargetDegree`, `EnclosureNote`, `EnclosureNoteRole`.
  - Функции: `resolveEnclosure(targetPc, type, key)`, `randomEnclosureType(rng)`.
  - Реэкспорт через `music-core/src/chords/index.ts`.
- **Приоритет:** P0
- **Зависимости:** `SCALE_INTERVALS`, `keyToPitchClass`, `spellPitchClass`, `chordDegreeToScale`.

### 3.5. Отображение опеваний в карточках

- **Проблема:** `PracticeBar` и `CardDisplay` не умеют показывать ступень и ноты оборота.
- **Описание:**
  - Расширить `PracticeBar` полями `enclosureDegree`, `enclosureType`, `enclosureNotes`.
  - В `CardDisplay` добавить ветку отображения опевания:
    - строка 1: крупный символ аккорда;
    - строка 2: `Ступень N`;
    - строка 3: ноты оборота, целевая нота выделена accent-цветом/рамкой.
- **Приоритет:** P0
- **Зависимости:** 3.3

### 3.6. Сохранение настроек

- **Проблема:** последние параметры опеваний должны сохраняться между сессиями.
- **Описание:** расширить `UserSettingsDTO.practiceCards` полями `lastExerciseType` (добавить `'embellishments'`), `lastEnclosureType`, `lastEnclosureDegrees`, `lastEnclosureOctaves`. Читать и сохранять их в `ExerciseWizard`.
- **Приоритет:** P1
- **Зависимости:** 3.2

## 5. Не войдёт в эту версию (Out of Scope)

- ❌ MIDI-оценка опеваний — отложено до Ф5.
- ❌ Секвенции — остаются disabled.
- ❌ Double enclosure, смешанные комбинации.
- ❌ Расширенные ступени `9, 11, 13` как цель.
- ❌ Такт на каждую выбранную ступень в режиме `over-chords` (в MVP одна случайная ступень на аккорд).
- ❌ Ноты на нотоносце и визуализация на клавиатуре/грифе.
- ❌ Пользовательские типы опеваний.

## 6. Риски и допущения

| ID | Риск | Решение |
| --- | --- | --- |
| R1 | Расчёт диатонических соседей требует лада — для нестандартных аккордов может быть неоднозначно | Использовать `chordDegreeToScale(degree, quality)` — детерминированный маппинг. Для альтераций — fallback на хроматические обороты. |
| R2 | При множестве ступеней пользователь не понимает, почему ступени случайны | В UI пояснение: «Выберите одну ступень для систематической отработки». |
| R3 | `parseChord` не возвращает массив ступеней `1/3/5/7` | Для MVP вычислять ступени по `quality + root` внутри `music-core/enclosures.ts`. Расширения `9/11/13` — P2. |
| R4 | Октавная привязка нот может выходить за удобный регистр | Генератор выбирает октаву относительно тоники аккорда; `octaves: 1 \| 2` для диапазона. |
| R5 | Карточка может быть перегружена | Компактный 3-строчный лейаут; на мобильных форсировать `mode='current'`. |
| R6 | Размещение `enclosures.ts` в `music-core` добавляет зависимость | `music-core` зависит только от stdlib + `@jazz/shared`; новые функции не нарушают boundaries. |

## 7. Метрики успеха

- Плитка «Опевания» активна и не показывает «скоро».
- Пользователь проходит wizard за <60 секунд и запускает упражнение.
- Карточка отображает 3 строки: аккорд → `Ступень N` → ноты оборота (цель выделена).
- `unified`-режим: на тоническом аккорде строятся обороты для каждой выбранной ступени.
- `over-chords`-режим: на каждый аккорд — один такт с одной случайной ступенью из выбранных.
- `EnclosureType === 'all'` — тип оборота меняется случайно на каждый такт.
- `playRandomly` — случайная тональность на каждый цикл прогрессии.
- Аккомпанемент, метроном, счётчик тактов работают корректно.
- Настройки сохраняются в `UserSettingsDTO`.
- `typecheck` + `lint` (boundaries) + `test` — зелёные.
- Существующие тесты `scaleExercise` и `chordExercise` не затронуты.
- Новые тесты `music-core/enclosures.test.ts` покрывают все 6 типов.
- Новые тесты `practice-cards/enclosureExercise.test.ts` покрывают `unified`, `over-chords`, `all`, `playRandomly`.
