**Дата:** 2026-06-17
**Горизонт:** три фазы (v1.0 → v1.1 → v1.2)
**Статус:** 🟡 Черновик (итерация 1)

---

## 1. Резюме (Executive Summary)

Раздел Theory — это **структурированная база знаний по джазовой теории** с глубокой интеграцией практики. В отличие от пассивных справочников (musictheory.net, Teoria), Jazz Trainer делает теорию **активной**: каждый теоретический блок сопровождается звуком, визуализацией нот на стане, встроенным мини-тренажёром и проверкой через игру на виртуальной/MIDI-клавиатуре.

Ключевая инновация — **лекционный движок (Lecture Engine)**: типизированный фреймворк, позволяющий создавать десятки лекций как отдельные плагины, с единым форматом блоков (текст → ноты → звук → практика → проверка). Контент-менеджер описывает лекцию как структуру данных на TypeScript — рендеринг унифицирован.

Лекции разбиты на **логические блоки** (темы), выстроены в **дорожную карту обучения** (как уровни в игре), содержат **босс-челленджи** в конце каждого блока (сложные практические задания с интерактивными эффектами). Прохождение накапливает **бонусные баллы** (экономика — в отдельном `MONETIZATION_VISION.md`).

## 2. Конкурентный контекст

### 2.1. Как подают теорию конкуренты

| Продукт             | Формат подачи          | Визуализация нот         | Звук         | Практика                    | Проверка              |
| ------------------- | ---------------------- | ------------------------ | ------------ | --------------------------- | --------------------- |
| **musictheory.net** | Статический справочник | Нотный стан (растр)      | MIDI-пианино | Отдельные упражнения        | Multiple choice       |
| **Teoria**          | Статьи + упражнения    | Нотный стан              | MIDI         | Отдельные упражнения        | Распознавание на слух |
| **EarMaster**       | Уроки (desktop)        | Нотный стан              | Да           | Встроенная (MIDI/голос)     | Да, адаптивная        |
| **Tenuto**          | Справочник (iOS)       | Нотный стан + клавиатура | Нет          | Упражнения на идентификацию | Визуальные тесты      |
| **ToneGym**         | Интерактивные уроки    | Нотный стан              | Да           | Да                          | Да, gamified          |
| **JazzPianoSchool** | Видео-лекции           | Нет (только видео)       | В плеере     | Нет встроенной              | Нет                   |
| **Open Studio**     | Видео-курсы            | Ноты в PDF               | В плеере     | Нет встроенной              | Нет                   |

### 2.2. Пробелы, которые закрывает Jazz Trainer

1. **Никто не объединяет теорию и практику в одном окне.** У всех теория и упражнения — разные экраны. Jazz Trainer: читаешь про опевания → сразу пробуешь на клавиатуре → система проверяет.

2. **Никто не подсвечивает ноты на стане синхронно со звуком.** Звучит нота → она подсвечивается на нотном стане. Это критически важно для связи «слух ↔ зрение».

3. **Ни у кого нет кнопки «Отработать это в тренажёре».** Прочитал про гамму → одна кнопка → полноценный тренажёр с аккомпанементом. Такой тесной интеграции нет ни у кого.

4. **Никто не делает проверку знаний через игру на инструменте.** Все используют multiple choice или распознавание на слух. Jazz Trainer: «Сыграй b7 в G7» → играешь на MIDI → система оценивает.

5. **Никто не предлагает единую дорожную карту обучения джазу.** Существующие сервисы — разрозненные справочники. Jazz Trainer — путь от «что такое аккордовые звуки» до «импровизация по блюзу».

## 3. Лекционный движок (Lecture Engine) — ядро формата

### 3.1. Философия: «See → Hear → Do → Verify»

Каждая лекция строится по 6-шаговому циклу, проверенному в музыкальной педагогике:

```
1. КОНТЕКСТ   → Зачем это нужно, где применяется, музыкальный пример
2. ТЕОРИЯ     → Объяснение концепции: текст + ноты + схемы + клавиатура
3. ЗВУК       → Проигрывание примера с синхронной подсветкой нот на стане
4. ПРАКТИКА   → Встроенный мини-тренажёр: сыграй на виртуальной/MIDI-клавиатуре
5. ПРОВЕРКА   → Активная проверка (игрой, а не кликом): система оценивает точность
6. ДАЛЬШЕ     → Кнопка перехода в полный тренажёр или к следующей лекции
```

**Почему это эффективно:**

- **Мультимодальность:** визуальная (ноты) + аудиальная (звук) + кинестетическая (игра) — задействованы все каналы восприятия.
- **Немедленная практика:** теория без практики забывается через 20 минут (кривая Эббингауза). Здесь практика — следующий шаг, а не отдельный экран.
- **Активная проверка:** игра на инструменте требует реального понимания (в отличие от угадывания в multiple choice).

### 3.2. Формат лекции: `LectureDefinition`

Лекция описывается как **типизированная структура данных** в TypeScript — это делает создание массовым и типобезопасным:

```ts
// packages/plugin-sdk/src/lecture-engine/types.ts

interface LectureDefinition {
  meta: LectureMeta;
  sections: LectureSection[];
}

interface LectureMeta {
  id: string; // 'theory.chord-tones'
  title: string; // 'Аккордовые звуки'
  topic: TopicId; // 'chord-tones' | 'approach-notes' | ...
  level: 1 | 2 | 3 | 4 | 5; // Уровень сложности (как в игре)
  duration: number; // Примерное время прохождения (мин)
  prerequisites: string[]; // ID лекций-пререквизитов
  bonusPoints: number; // Баллы за полное прохождение
  tags: string[]; // ['начинающий', 'гармония', 'импровизация']
}

type LectureBlock =
  | { type: 'text'; content: string } // MD-текст
  | { type: 'image'; src: string; caption?: string } // Иллюстрация
  | { type: 'diagram'; mermaid: string } // Mermaid-схема
  | { type: 'sheet-music'; notes: string; playback: PlaybackConfig } // Нотный стан OSMD
  | { type: 'keyboard'; highlight: Note[]; label?: string } // Вирт. клавиатура
  | { type: 'audio'; src: string; waveform?: boolean } // MP3/M4A плеер
  | { type: 'video'; src: string; poster?: string } // Видео-плеер
  | { type: 'mini-trainer'; exercise: MiniExercise } // Мини-тренажёр
  | { type: 'quiz'; quiz: ActiveQuiz } // Активная проверка
  | { type: 'divider' } // Разделитель
  | { type: 'callout'; kind: 'tip' | 'warning' | 'info'; content: string };

interface LectureSection {
  id: string; // 'context', 'theory', 'sound', ...
  title: string; // '1. Контекст', '2. Теория', ...
  blocks: LectureBlock[];
}
```

### 3.3. Компоненты Lecture Engine

Лекционный движок рендерит `LectureDefinition` через универсальный компонент `LectureRenderer`:

```
┌─ LectureRenderer ──────────────────────────────────────────┐
│  ┌─ LectureHeader: название, уровень, длительность, теги ─┐│
│  ├─ Section: "1. Контекст" ───────────────────────────────┤│
│  │  ├─ Block[text]     → Markdown-рендерер                ││
│  │  └─ Block[callout]  → Цветной блок с иконкой           ││
│  ├─ Section: "2. Теория" ────────────────────────────────┤│
│  │  ├─ Block[text]     → Markdown-рендерер                ││
│  │  ├─ Block[sheet-music] → OSMD-компонент                 ││
│  │  └─ Block[keyboard] → VirtualKeyboard                  ││
│  ├─ Section: "3. Звук" ──────────────────────────────────┤│
│  │  └─ Block[sheet-music] → OSMD с playback + подсветкой   ││
│  ├─ Section: "4. Практика" ──────────────────────────────┤│
│  │  └─ Block[mini-trainer] → MiniTrainer                  ││
│  ├─ Section: "5. Проверка" ──────────────────────────────┤│
│  │  └─ Block[quiz]     → ActiveQuiz (игровая проверка)     ││
│  ├─ Section: "6. Дальше" ────────────────────────────────┤│
│  │  ├─ Button: "Отработать в тренажёре" → /trainer/...    ││
│  │  └─ Button: "Следующая лекция" → /theory/...           ││
│  └─ LectureFooter: прогресс-бар, баллы                     ││
└───────────────────────────────────────────────────────────┘
```

**Где живут компоненты:**

| Компонент         | Расположение              | Зависимости                   |
| ----------------- | ------------------------- | ----------------------------- |
| `LectureRenderer` | `packages/ui/` (общий UI) | React                         |
| `OSMDSheetMusic`  | `packages/ui/`            | OSMD (новый npm-пакет)        |
| `VirtualKeyboard` | `packages/ui/`            | Tone.js (уже есть)            |
| `MiniTrainer`     | `packages/ui/`            | `midiEval` (из `music-core`)  |
| `ActiveQuiz`      | `packages/ui/`            | `midiEval`, `TransportEngine` |
| `AudioPlayer`     | `packages/ui/`            | Tone.js / Web Audio           |
| `VideoPlayer`     | `packages/ui/`            | `<video>`                     |

### 3.4. Пример: лекция «Аккордовые звуки» (Chord Tones)

```ts
// packages/plugins/theory-chord-tones/src/lecture.ts

export const chordTonesLecture: LectureDefinition = {
  meta: {
    id: 'theory.chord-tones',
    title: 'Аккордовые звуки',
    topic: 'chord-tones',
    level: 1,
    duration: 15,
    prerequisites: [],
    bonusPoints: 50,
    tags: ['начинающий', 'гармония'],
  },
  sections: [
    {
      id: 'context',
      title: '1. Контекст',
      blocks: [
        {
          type: 'text',
          content: `Аккордовые звуки — фундамент джазовой импровизации...`,
        },
        {
          type: 'callout',
          kind: 'tip',
          content: 'В джазе 80% импровизации — обыгрывание аккордовых звуков.',
        },
      ],
    },
    {
      id: 'theory',
      title: '2. Теория',
      blocks: [
        {
          type: 'text',
          content: `Любой септаккорд состоит из 4-х звуков: тоника (1), терция (3), квинта (5), септима (7)...`,
        },
        {
          type: 'sheet-music',
          notes: 'Cmaj7: C4 E4 G4 B4',
          playback: { tempo: 80, loop: true },
        },
        {
          type: 'keyboard',
          highlight: ['C4', 'E4', 'G4', 'B4'],
          label: 'Cmaj7 — аккордовые звуки',
        },
        {
          type: 'text',
          content: `### Типы септаккордов и их звуки\n\n| Тип | Формула | Пример (C) |\n|-----|---------|------------|\n| maj7 | 1-3-5-7 | C-E-G-B |\n| m7 | 1-♭3-5-♭7 | C-E♭-G-B♭ |\n| 7 | 1-3-5-♭7 | C-E-G-B♭ |\n| m7♭5 | 1-♭3-♭5-♭7 | C-E♭-G♭-B♭ |`,
        },
      ],
    },
    {
      id: 'sound',
      title: '3. Звук',
      blocks: [
        {
          type: 'sheet-music',
          notes: 'Cmaj7 ascending: C4 E4 G4 B4 | Dm7 ascending: D4 F4 A4 C5',
          playback: {
            tempo: 60,
            highlightNotes: true, // подсветка нот при проигрывании
            instrument: 'piano',
          },
        },
      ],
    },
    {
      id: 'practice',
      title: '4. Практика',
      blocks: [
        {
          type: 'mini-trainer',
          exercise: {
            type: 'play-arpeggio',
            chords: ['Cmaj7', 'Dm7', 'G7'],
            input: 'midi', // виртуальная клавиатура + MIDI
            feedback: 'note-by-note', // зелёный — правильно, красный — ошибка
          },
        },
      ],
    },
    {
      id: 'assessment',
      title: '5. Проверка',
      blocks: [
        {
          type: 'quiz',
          quiz: {
            type: 'play-the-note',
            questions: [
              { prompt: 'Сыграй терцию аккорда Cmaj7', expectedNote: 'E4' },
              { prompt: 'Сыграй септиму аккорда Dm7', expectedNote: 'C5' },
            ],
            input: 'midi',
          },
        },
      ],
    },
    {
      id: 'next',
      title: '6. Дальше',
      blocks: [
        {
          type: 'callout',
          kind: 'info',
          content: 'Отработай арпеджио во всех 12 тональностях в полном тренажёре.',
        },
        // Кнопка «Открыть тренажёр арпеджио» — рендерится LectureRenderer
        // автоматически на основе meta.topic
      ],
    },
  ],
};

## 5. Фаза 1 — v1.0: Lecture Engine MVP

**Цель:** Запустить лекционный движок с базовыми компонентами и 6 первыми лекциями, покрывающими фундамент джазовой теории.

### 5.1. Новые функции

#### 5.1.1. Лекционный движок (Lecture Engine)

- **Проблема:** Нет единого формата для создания и отображения теоретических материалов. Каждый плагин theory-\* — заглушка.
- **Описание:** Типизированный фреймворк `LectureDefinition` + универсальный `LectureRenderer`. Плагин-лекция экспортирует определение → рендерится единообразно.
- **Конкуренты:** Ни у кого нет программируемого лекционного движка. Teoria и musictheory.net — статические HTML-страницы. ToneGym — проприетарный движок без возможности расширения.
- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** `packages/ui/` (компоненты) + `packages/plugin-sdk/` (типы `LectureDefinition`)
- **Зависимости:** OSMD (новый npm-пакет), Tone.js (уже есть), `midiEval` (уже есть)

**Блоки v1.0:**

| Блок                           | Статус v1.0 | Описание                                      |
| ------------------------------ | ----------- | --------------------------------------------- |
| `text`                         | ✅          | Markdown-рендеринг (react-markdown)           |
| `sheet-music`                  | ✅          | OSMD: рендер нот + playback с подсветкой      |
| `keyboard`                     | ✅          | Виртуальная клавиатура с подсветкой нот       |
| `mini-trainer` (play-arpeggio) | ✅          | «Сыграй арпеджио» через вирт./MIDI-клавиатуру |
| `quiz` (play-the-note)         | ✅          | «Сыграй ноту X» — проверка через `midiEval`   |
| `callout`                      | ✅          | Цветные блоки tip/warning/info                |
| `divider`                      | ✅          | Разделитель секций                            |

**Блоки, отложенные на v1.1:**

| Блок                         | Причина                              |
| ---------------------------- | ------------------------------------ |
| `image`                      | Нужна загрузка и хостинг изображений |
| `diagram` (mermaid)          | Нужен mermaid-рендерер               |
| `audio` (mp3/m4a)            | Нужен аудио-плеер + хостинг файлов   |
| `video`                      | Нужен видео-плеер + хостинг/YouTube  |
| `mini-trainer` (другие типы) | Только play-arpeggio в v1.0          |
| `quiz` (другие типы)         | Только play-the-note в v1.0          |

#### 5.1.2. OSMD-интеграция: нотный стан с подсветкой

- **Проблема:** Нет визуализации нот на нотоносце. Пользователь видит только названия нот и клавиатуру — этого мало для музыкантов, читающих ноты.
- **Описание:** Компонент `OSMDSheetMusic` оборачивает OpenSheetMusicDisplay. Принимает строку нот (MusicXML или простой ABC-подобный формат), рендерит нотоносец. При проигрывании через Tone.js подсвечивает текущую ноту.
- **Конкуренты:** Flat и Soundslice имеют нотный стан, но без синхронной подсветки при проигрывании теории. Musictheory.net — статический растр.
- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** `packages/ui/` → `OSMDSheetMusic.tsx`
- **Зависимости:** `opensheetmusicdisplay` (npm), Tone.js (для playback)

#### 5.1.3. Виртуальная клавиатура

- **Проблема:** Нет интерактивной клавиатуры для визуализации и ввода нот без MIDI-устройства.
- **Описание:** Компонент `VirtualKeyboard` — горизонтальная фортепианная клавиатура (2–3 октавы). Поддерживает: подсветку нот (передаётся массив `Note[]`), клик для ввода (мышь/тач), анимацию нажатия, подписи нот (опционально). Используется и в лекциях (подсветка), и в мини-тренажёрах (ввод).
- **Конкуренты:** ToneGym имеет виртуальную клавиатуру. EarMaster имеет. Teoria/musictheory.net — нет. Tenuto — только статическая.
- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** `packages/ui/` → `VirtualKeyboard.tsx`
- **Зависимости:** Tone.js (синтез звука при клике), Web Audio API

#### 5.1.4. Мини-тренажёр «Сыграй арпеджио»

- **Проблема:** После прочтения теории пользователю нужно немедленно попробовать — но идти в отдельный тренажёр — трение.
- **Описание:** Встроенный в лекцию мини-тренажёр. Показывает аккорд (например, «Cmaj7»), пользователь играет арпеджио (C→E→G→B) на виртуальной клавиатуре или MIDI. Система через `midiEval` подсвечивает: зелёный = правильная нота, красный = ошибка. После правильного прохождения — следующий аккорд.
- **Конкуренты:** Никто не встраивает тренажёр прямо в теорию. EarMaster — отдельные упражнения. ToneGym — отдельные «геймы».
- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** `packages/ui/` → `MiniTrainer.tsx`
- **Зависимости:** `midiEval` (из `music-core`), `VirtualKeyboard`, `webmidi-adapter`

#### 5.1.5. Активная проверка «Сыграй ноту»

- **Проблема:** Традиционные квизы (multiple choice) не проверяют реальное понимание — можно угадать. Для музыканта критично знать ноты на инструменте, а не в теории.
- **Описание:** `ActiveQuiz` типа `play-the-note`: система спрашивает «Сыграй терцию Cmaj7» → пользователь играет ноту на виртуальной/MIDI-клавиатуре → `midiEval` проверяет. Мгновенная обратная связь: правильно/неправильно + правильный ответ на клавиатуре. Серия из 5–10 вопросов, результат в конце.
- **Конкуренты:** Ни у кого нет проверки через игру. EarMaster — closest, но это отдельный режим тренировки, не встроенный в лекцию.
- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** `packages/ui/` → `ActiveQuiz.tsx`
- **Зависимости:** `midiEval`, `VirtualKeyboard`

#### 5.1.6. Быстрый переход в тренажёр

- **Проблема:** Пользователь изучил концепцию в лекции, хочет отработать в полноценном тренажёре — но нужно вручную искать тренажёр, настраивать параметры.
- **Описание:** Кнопка в конце лекции (секция «Дальше»), которая автоматически открывает релевантный тренажёр с предзаполненными параметрами из лекции. Например: лекция про арпеджио Cmaj7 → кнопка «Отработать во всех тональностях» → `/practice-cards?type=arpeggio&chord=Cmaj7`.
- **Конкуренты:** Ни у кого нет.
- **Приоритет:** **P1** (важно, но не блокирует MVP)
- **Плагин / Модуль:** Логика в `LectureRenderer`, роутинг в `plugin-host`
- **Зависимости:** Существующие тренажёры (`practice-cards`, `ear-training`, `rhythm-drills`)

#### 5.1.7. Первые 6 лекций

- **Проблема:** Раздел теории — заглушки. Нет контента.
- **Описание:** 6 плагинов-лекций, покрывающих джазовый фундамент. Каждый плагин экспортирует `LectureDefinition`, регистрируется в `plugin-registry`.

| #   | Плагин                  | Тема                                       | Уровень | Баллы |
| --- | ----------------------- | ------------------------------------------ | ------- | ----- |
| 1   | `theory-chord-tones`    | Аккордовые звуки (Chord Tones)             | 1       | 50    |
| 2   | `theory-approach-notes` | Опевания аккордовых тонов (Approach Notes) | 1       | 50    |
| 3   | `theory-arpeggios`      | Арпеджио (Arpeggios)                       | 1       | 50    |
| 4   | `theory-rhythm`         | Ритм в импровизации                        | 2       | 75    |
| 5   | `theory-groove`         | Грув и фразировка                          | 2       | 75    |
| 6   | `theory-blues`          | Блюз                                       | 2       | 100   |

**Детализация тем:**

1. **Аккордовые звуки:** Что такое chord tones (1-3-5-7). Типы септаккордов (maj7, m7, 7, m7♭5). Звучание каждого. Как находить на клавиатуре.

2. **Опевания:** Хроматические и диатонические опевания (approach notes). Upper/lower neighbor. Enclosures. Практика: опевание терции и септимы.

3. **Арпеджио:** Восходящие/нисходящие арпеджио септаккордов. Арпеджио с опеванием. Арпеджио через октаву. Практика: игра арпеджио под аккомпанемент.

4. **Ритм в импровизации:** Восьмые, триоли, шестнадцатые. Свинг. Синкопы. Игра с метрономом на 2 и 4. Практика: ритмические паттерны на одном звуке.

5. **Грув и фразировка:** Что такое грув. Артикуляция (legato, staccato, акценты). Джазовая фразировка (опережение/запаздывание). Практика: фразировка арпеджио под ритм-секцию.

6. **Блюз:** 12-тактовый блюз. Блюзовая гамма. Блюзовые ноты (♭3, ♭5, ♭7). Блюзовые обороты (turnarounds). Практика: импровизация по блюзовому квадрату.

- **Приоритет:** **P0** (MVP)
- **Плагин / Модуль:** 6 новых плагинов `packages/plugins/theory-*/`
- **Зависимости:** Lecture Engine, OSMD, VirtualKeyboard, MiniTrainer

### 7.1. Прогресс и баллы

- **Проблема:** Пользователь не видит своего прогресса. Нет мотивации проходить лекции последовательно.
- **Описание:**

  - Дорожная карта обучения визуализируется как дерево: пройденные лекции 🟢, доступные 🟡, заблокированные 🔴 (требуют пререквизитов).

### 7.3. Босс-челленджи

- **Проблема:** Нет кульминационных моментов в обучении. Нет проверки комплексных навыков.
- **Описание:** В конце каждого логического блока — **босс-челлендж**. Это сложное практическое задание с:
  - **Таймером** (ограничение по времени добавляет adrenaline)
  - **Полосой здоровья** (3 попытки = 3 сердца, ошибка = минус сердце)
  - **Визуальными эффектами** (экран темнеет при ошибке, вспышка при успехе, пульсация при низком здоровье)
  - **Музыкальным сопровождением** (аккомпанемент играет, нужно импровизировать поверх)
  - **Счётом** (оценивается точность нот, ритма, артикуляции)
  - **Рейтингом** (1–3 звезды по результату)

**Пример: Босс «Блюзовый квадрат»**

```
┌─ 🦍 БОСС: Блюзовый квадрат ────────────────────────────────┐
│                                                             │
│  Сыграй 3 chorus блюза в C. Используй блюзовую гамму       │
│  и аккордовые звуки.                                        │
│                                                             │
│  ♥ ♥ ♥   ❌ 2/3 попытки              ⏱ 02:34 / 05:00      │
│                                                             │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Такт 5: C7  |  Ожидается: C блюзовая гамма          │  │
│  │  Вы сыграли: C Eb Gb G Bb C Bb G                     │  │
│  │  🟢🟢🟢🟢🟢🟢🟢🟢   (100% точных нот)               │  │
│  │  Точность ритма: ████████░░ 82%                      │  │
│  └──────────────────────────────────────────────────────┘  │

## 11. Метрики успеха

### Фаза 1 (v1.0)

- **Контент:** 6 лекций опубликовано, все с нотным станом, звуком и мини-тренажёром
- **Проходимость:** >60% пользователей, начавших лекцию, доходят до секции «Проверка»
- **Практика:** >40% проходят мини-тренажёр хотя бы раз
- **Среднее время на лекцию:** 10–20 минут (соответствует заявленной `duration`)
- **Ошибки:** <5% багов в OSMD-подсветке и `midiEval`

### Фаза 2 (v1.1)

- **Контент:** 18 лекций опубликовано (6 + 12 новых)
- **Возвращаемость:** >30% пользователей возвращаются к следующей лекции в течение 7 дней
- **Аудио/видео:** >20% лекций содержат реальные аудио-примеры
