# EXERCISE VISION — Интерактивные упражнения Jazz Trainer

**Дата:** 2026-07-19
**Горизонт:** 1 версия (4–6 недель)
**Статус:** 🟡 Черновик — ожидает подтверждения

## 1. Резюме (Executive Summary)

Новый раздел тренировки Jazz Trainer: **интерактивные упражнения** — практика джазовых навыков с живым аккомпанементом и обратной связью. Пользователь выбирает тип упражнения из каталога, настраивает параметры, играет на своём инструменте и получает оценку точности (где применимо). Система объединяет пять направлений развития музыканта: ритмические навыки, нотную грамоту, структуру и функцию аккордов, мелодический и гармонический слух, а также мелодические диктанты.

Ключевая ценность: **единая среда для всех видов музыкальной практики** — от чтения нот до распознавания прогрессий на слух, с общим движком аккомпанемента (Bass, Drums, Piano, Rhodes, Guitar), единым UX-паттерном «мастер → практика → результат» и сквозной статистикой.

**Принципиальное отличие от предыдущей версии (EXERCISE VISION v0.1):** упражнения — это не только «карточки». Карточки (`practice-cards`) — лишь один из **режимов отображения** внутри экосистемы упражнений. Другие режимы: слуховой (диктанты, распознавание), визуально-нотный (чтение с листа), ритмический (tap-along, repeat-after-me), теоретический (квизы, конструкторы).

**MVP:** 5 типов упражнений распределены по трём приоритетам:
- **P0 (MVP ядро):** ритмические упражнения, распознавание аккордов на слух, чтение нот.
- **P1 (MVP расширение):** аккордовые прогрессии (карточки), гаммы (карточки).
- **P2 (вторая итерация):** мелодические диктанты, гармонический слух.

Существующие плагины (`ear-training`, `rhythm-drills`, `chord-quiz`, `progression-recognition`, `practice-cards`) встраиваются в единую систему через общие компоненты: мастер настройки, экран результата, движок упражнений.

## 3. Новые функции (Features)

### 3.1. Единый мастер настройки упражнения (Exercise Configuration Wizard v2)

- **Проблема:** Пользователь должен быстро выбрать тип упражнения и настроить параметры через единый интерфейс, независимо от типа.
- **Описание:** Общий 3-шаговый мастер, переиспользуемый всеми типами упражнений:

  **Шаг 1 — Выбор категории и типа упражнения:**
  Пять категорий-плиток с иконками, каждая раскрывается в подтипы:

  | Категория | Иконка | Подтипы |
  |-----------|--------|---------|
  | 🥁 Ритмические навыки | 🥁 | Чтение ритма, Повтор ритма, Свинг-пульсация, Полиритмия |
  | 🎼 Нотная грамота | 🎼 | Ноты на нотоносце, Ключевые знаки, Аккорды в нотации, Ритм в нотации |
  | 🎸 Структура и функция аккордов | 🎸 | Состав аккорда, Функция аккорда, Аккордовые прогрессии (карточки), Voice Leading, Гаммы по прогрессии |
  | 👂 Мелодический и гармонический слух | 👂 | Интервалы, Качества аккордов, Прогрессии, Лады, Обращения, Басовый слух |
  | ✍️ Мелодические диктанты | ✍️ | Одноголосный, Двухголосный, Ритмический, Джазовая фраза |

  Подтипы с ⏳ — запланированы на следующие версии, показаны с пометкой «Скоро».

  **Шаг 2 — Параметры упражнения (зависят от подтипа):**
  Универсальные параметры (для всех типов):
  - **Темп:** слайдер 40–300 BPM
  - **Тональность:** выбор из 12 тональностей
  - **Источник:** встроенный набор / генератор / ручной DSL-ввод (где применимо)
  - **Сложность:** начальный / средний / продвинутый (влияет на набор аккордов/интервалов/ритмов)
  - **Количество:** 5 / 10 / 20 / ∞ (вопросов/тактов)

  Специфичные параметры (для каждого подтипа) — см. соответствующие разделы 3.2–3.6.

  **Шаг 3 — Инструкция и старт:**
  Превью того, что будет в упражнении + кнопка **«▶ Старт»**.

- **Конкуренты:** EarMaster имеет пошаговый выбор, но жёстко привязан к курсу. Никто не даёт свободного конфигурирования.
- **Приоритет:** P0 (MVP — общий компонент для всех типов)
- **Усиливает / Новая категория:** `practice` (общий компонент `ExerciseWizard` в `@jazz/plugin-sdk` или shared UI)
- **Зависимости:** Существующие плагины тренировки должны поддерживать общий конфиг-интерфейс.

### 3.2. Ритмические упражнения (Rhythm Exercises)

- **Проблема:** Чувство ритма — фундаментальный навык, без которого невозможна джазовая импровизация. Существующий `rhythm-drills` покрывает только базовое «повтори ритм». Нужна расширенная программа: чтение ритмической нотации, свинг-пульсация, полиритмия.
- **Описание:** Четыре подтипа ритмических упражнений:

  **А) Чтение ритма (Rhythm Reading):**
  На экране — нотный стан с ритмическим паттерном (одна нота на разных длительностях). Пользователь должен простучать ритм на MIDI-клавише/пэде в заданном темпе. Оценка: `midiEval` сравнивает тайминг ударов с эталоном, отображает отклонение в миллисекундах.

  **Б) Повтор ритма (Rhythm Repeat — улучшение существующего `rhythm-drills`):**
  Прослушивание ритмического паттерна → повтор на MIDI. Добавляется: визуальная нотация прослушанного паттерна (после ответа), уровни сложности (простые длительности → синкопы → свинг → латинские рисунки).

  **В) Свинг-пульсация (Swing Feel):**
  Специфично для джаза: тренировка триольного ощущения восьмых. Пользователь играет простой паттерн (например, четверти), а система оценивает точность свингового грува (отношение длительностей восьмых в триоли 2:1).

  **Г) Полиритмия (Polyrhythms):**
  2 против 3, 3 против 4, 5 против 4. Одна рука отбивает один ритм, вторая — другой. MIDI-оценка каждого голоса отдельно.

- **Конкуренты:** EarMaster имеет чтение ритма и повтор. Свинг-пульсация и полиритмия с MIDI-оценкой — уникальные фичи для джазового тренажёра.
- **Приоритет:** P0 (ядро MVP) для А, Б; P1 для В; P2 для Г
- **Усиливает плагин:** `rhythm-drills` (расширение)
- **Зависимости:** `music-core/audio` (MIDI-оценка), `webmidi-adapter`

### 3.3. Нотная грамота (Notation Exercises)

- **Проблема:** Многие джазовые музыканты (особенно самоучки) слабо читают ноты. Существующие инструменты (Tenuto, musictheory.net) не имеют джазового контекста и звука.
- **Описание:** Четыре подтипа:

  **А) Ноты на нотоносце (Note Identification):**
  На экране — нота на нотоносце. Пользователь должен сыграть её на MIDI-клавиатуре. Оценка: правильно/неправильно + визуальная подсветка клавиши на экранной клавиатуре. Уровни: без ключевых знаков → с ключевыми знаками (все 12 тональностей) → с альтерациями.

  **Б) Ключевые знаки (Key Signatures):**
  Показан ключевой знак (например, 3 диеза). Пользователь определяет тональность (A-dur / fis-moll) и играет гамму. Оценка: правильная тональность + точность гаммы.

  **В) Аккорды в нотации (Chord Notation):**
  На нотоносце — аккорд (например, Cmaj7 в тесном расположении). Пользователь определяет символ аккорда (текстовый ввод или выбор из вариантов). Джазовая специфика: аккорды с надстройками (9, 11, 13), альтерации (b9, #11).

  **Г) Ритм в нотации (Rhythm Notation):**
  Комбинация чтения ритма (§3.2-А) и нот: на нотоносце — мелодия. Пользователь играет в заданном темпе. Оценка: точность нот + точность ритма.

- **Конкуренты:** Tenuto и musictheory.net делают А и Б, но без звука, без MIDI и без джазового контекста. В, Г — уникальны.
- **Приоритет:** P0 (А), P1 (Б, В), P2 (Г)
- **Усиливает плагин / Новая категория:** Новый плагин `notation-exercises` (категория `practice`)
- **Зависимости:** `music-core/audio` (MIDI-оценка), `webmidi-adapter`, визуальный нотоносец (новый компонент)

### 3.4. Структура и функция аккордов (Chord Structure & Function)

- **Проблема:** Понимание состава аккордов и их функции в тональности — ключевой навык для импровизации и компинга. Существующий `chord-quiz` покрывает только состав. Нужна практика функции и практика игры.
- **Описание:** Пять подтипов:

  **А) Состав аккорда (Chord Spelling — улучшение `chord-quiz`):**
  Показан символ аккорда (например, `Dm7b5`). Пользователь играет звуки аккорда на MIDI-клавиатуре. Оценка: `midiEval` сверяет набор нот. Добавляется: джазовые аккорды с надстройками (9, 11, 13), альтерации, обращения.

  **Б) Функция аккорда (Chord Function):**
  Задана тональность (например, C-dur). Показан аккорд — нужно определить его функцию (тоника / субдоминанта / доминанта) или ступень (I, ii, V и т.д.). Для продвинутого уровня: побочные доминанты, тритоновые замены, модальный обмен.

  **В) Аккордовые прогрессии — карточки (Chord Progression Cards — рефрейминг `practice-cards`):**
  Визуальный «телесуфлёр»: карточки с символами аккордов сменяются по тактам (1, 2 или 3 карточки одновременно) под живой аккомпанемент. Пользователь играет аккорды на своём инструменте. **Без MIDI-оценки** (аккомпанемент + визуальная подсказка). Источник: встроенные паттерны (ii–V–I, turnaround, rhythm changes и др.) / генератор / ручной DSL-ввод.

  **Г) Voice Leading:**
  Дана последовательность аккордов (например, `Dm7 G7 Cmaj7`). Пользователь должен сыграть их с плавным голосоведением (минимальное движение голосов). Оценка: `midiEval` проверяет, что все ноты аккорда присутствуют и движение каждого голоса ≤ 2 полутонов.

  **Д) Гаммы по прогрессии (Scale over Changes — рефрейминг scale cards):**
  Для каждого аккорда прогрессии автоматически определяется соответствующий лад. Карточка показывает: символ аккорда + название лада (например, `Dm7 — D дорийский`). Пользователь играет гамму на каждый такт.

- **Конкуренты:** EarMaster — состав аккордов (квиз). iReal Pro — прогрессии (игра, без оценки). Voice Leading с автоматической проверкой — уникальная фича.
- **Приоритет:** P0 (А, Б), P1 (В, Д), P2 (Г)
- **Усиливает плагины:** `chord-quiz` (расширение до А, Б), `practice-cards` (рефрейминг в В, Д), новый `voice-leading-exercises` (Г)
- **Зависимости:** `music-core/audio` (CardDisplay, TransportEngine, midiEval), `music-core/generator`, `music-core/dsl`

### 3.5. Мелодический и гармонический слух (Ear Training)

- **Проблема:** Развитый слух — главный инструмент джазового музыканта. Существующий `ear-training` покрывает только интервалы, `progression-recognition` — только прогрессии. Нужна единая система, покрывающая все аспекты джазового слуха.
- **Описание:** Шесть подтипов, объединяющих и расширяющих существующие плагины:

  **А) Интервалы (Intervals — улучшение `ear-training`):**
  Прослушивание двух нот (последовательно или гармонически). Определить интервал — сыграть ответ на MIDI. Добавляется: интервалы в контексте тональности, хроматические интервалы, составные интервалы (> октавы).

  **Б) Качества аккордов (Chord Qualities):**
  Прослушивание аккорда — определить тип (maj7, m7, 7, m7b5, dim7, aug, mMaj7, 7sus4 и др.). Ответ: выбор из вариантов или игра на MIDI.

  **В) Прогрессии (Progressions — улучшение `progression-recognition`):**
  Прослушивание гармонической последовательности — определить тип (ii–V–I, I–IV–V, turnaround, блюзовый квадрат). Добавляется: прогрессии с побочными доминантами, тритоновыми заменами, модальным обменом.

  **Г) Лады (Modes):**
  Прослушивание гаммы в контексте аккорда — определить лад (дорийский, миксолидийский и т.д.). Джазовая специфика: гамма играется на фоне аккомпанемента (бас + аккорд), чтобы слышать характер лада.

  **Д) Обращения аккордов (Chord Inversions):**
  Прослушивание аккорда — определить обращение (основной вид, 1-е, 2-е, 3-е) и сыграть его.

  **Е) Басовый слух (Bass Hearing):**
  Прослушивание гармонической последовательности — определить движение баса (ступени: I–II–V–I или ноты: C–D–G–C). Ответ: сыграть басовую линию на MIDI.

- **Конкуренты:** EarMaster покрывает А, Б, В, но без джазового аккомпанемента. Г, Д, Е с джазовым контекстом — уникальны.
- **Приоритет:** P0 (А, Б), P1 (В, Г), P2 (Д, Е)
- **Усиливает плагины:** `ear-training` (расширение до А, Б, Д, Е), `progression-recognition` (расширение до В), новый `mode-ear-training` (Г)
- **Зависимости:** `music-core/audio` (midiEval, TransportEngine), `music-core/generator`, `webmidi-adapter`

### 3.6. Мелодические диктанты (Melodic Dictations)

- **Проблема:** Умение записать услышанную мелодию (диктант) — классический навык, обязательный в музыкальном образовании. Для джаза критично: транскрипция соло, запись идей. Никто не делает диктанты с джазовым контекстом и MIDI-вводом.
- **Описание:** Четыре подтипа:

  **А) Одноголосный диктант (Single-Voice Dictation):**
  Прослушивание короткой мелодии (2–4 такта) в джазовом стиле. Пользователь играет услышанное на MIDI-клавиатуре. Оценка: `midiEval` сравнивает ноты и ритм. Сложность: диатоника → хроматика → альтерации → джазовые фразы (bebop, блюзовые ноты). Количество прослушиваний: настраивается (3–5–10 раз).

  **Б) Двухголосный диктант (Two-Voice Dictation):**
  Прослушивание двухголосной фразы (мелодия + бас, или два голоса в интервалах). Пользователь играет оба голоса последовательно. Оценка: каждый голос отдельно.

  **В) Ритмический диктант (Rhythm Dictation):**
  Прослушивание ритмического паттерна (одна нота). Пользователь простукивает ритм. Визуализация: после ответа показывается ритмическая нотация.

  **Г) Джазовая фраза (Jazz Lick Dictation):**
  Прослушивание фразы в стиле bebop/hard-bop с характерными элементами (хроматические подходные ноты, опевания, блюзовые ноты, свинг). Пользователь играет фразу. После ответа — показ нот с анализом (какие ноты аккордовые, какие подходные).

- **Конкуренты:** EarMaster имеет одно- и двухголосные диктанты, но без джазового контекста. Джазовая фраза с анализом — уникальная фича.
- **Приоритет:** P2 (вторая итерация после MVP ядра)
- **Усиливает плагин / Новая категория:** Новый плагин `melodic-dictation` (категория `practice`)
- **Зависимости:** `music-core/audio` (midiEval, TransportEngine — для проигрывания диктанта), `music-core/generator` (для генерации мелодий)

### 3.7. Единый экран результата (Unified Results Screen)

- **Проблема:** Каждый тип упражнения сейчас показывает результаты по-своему. Нужен единый экран завершения с общей статистикой.
- **Описание:** После завершения любого упражнения — экран с:
  - **Сводка:** тип упражнения, длительность, количество вопросов/тактов
  - **Точность:** процент правильных ответов, график (если несколько попыток)
  - **Детализация:** список всех вопросов с результатами (правильно/неправильно, сыгранные ноты vs ожидаемые)
  - **Слабые места:** автоматическое выделение типов ошибок (например, «путаешь m7 и m7b5»)
  - **Кнопки:** «Повторить» / «Настроить заново» / «Следующее упражнение» (рекомендация)
- **Конкуренты:** EarMaster показывает детальную статистику. Tenuto — только счёт.
- **Приоритет:** P0 (MVP — общий компонент)
- **Усиливает / Новая категория:** Общий компонент в `@jazz/plugin-sdk` или `shared-ui`
- **Зависимости:** Все плагины практики должны использовать этот компонент.

### 3.8. Сквозная статистика и прогресс (Cross-Exercise Stats)

- **Проблема:** Пользователь не видит свой прогресс в целом — только в рамках одного типа упражнений.
- **Описание:**
  - Сохранение результатов всех упражнений в `exercise_history` (БД)
  - Дашборд прогресса: общее время практики, сильные/слабые направления, график точности по дням
  - «Ежедневная практика» (daily streak): автоматически подобранный набор из 3–5 упражнений на 15 минут
  - DTO: `ExerciseResultDTO` (тип, подтип, конфигурация, точность, длительность, timestamp)
- **Конкуренты:** EarMaster имеет статистику. Duolingo-подобный streak — уникален для музыкального EdTech.
- **Приоритет:** P1 (после MVP ядра)
- **Усиливает / Новая категория:** `apps/api` (новая таблица + API), личный кабинет пользователя
- **Зависимости:** 3.7 (Unified Results Screen), сохранение через `apiClient`

## 4. Улучшения существующих функций

### 4.1. Рефакторинг `practice-cards` → режим отображения «Карточки»

- `practice-cards` из отдельного плагина становится одним из **режимов отображения** внутри системы упражнений. Его конфигурация (CardDisplay, ExerciseRunner, генераторы) сохраняется, но встраивается в общий ExerciseWizard и Results Screen.
- Карточки используются в упражнениях типов: «Аккордовые прогрессии» (§3.4-В), «Гаммы по прогрессии» (§3.4-Д).

### 4.2. Рефакторинг `ear-training` → модули слуховых упражнений

- Интервалы (§3.5-А), обращения (§3.5-Д), басовый слух (§3.5-Е) — расширяют существующий `ear-training`.
- `ear-training` переиспользует общие компоненты ExerciseWizard и Results Screen.

### 4.3. Рефакторинг `rhythm-drills` → расширенная ритмическая программа

- Чтение ритма (§3.2-А), свинг-пульсация (§3.2-В), полиритмия (§3.2-Г) — расширяют существующий `rhythm-drills`.
- Визуализация ритмической нотации — новый компонент.

### 4.4. Рефакторинг `chord-quiz` → структура и функция аккордов

- Состав аккорда (§3.4-А) и функция аккорда (§3.4-Б) — расширяют `chord-quiz`.

### 4.5. Рефакторинг `progression-recognition` → слуховые упражнения

- Прогрессии (§3.5-В) и лады (§3.5-Г) — расширяют `progression-recognition`.

### 4.6. Интеграция пользовательских настроек

- Сохранять последние параметры **каждого типа упражнения** в `UserSettingsDTO` через `useSettings()`.
- Новые поля в `UserSettingsDTO` (предложение):
  ```ts
  exercisePreferences: {
    lastExerciseType?: string;   // e.g. 'rhythm.reading'
    lastConfig?: Record<string, unknown>;  // типобезопасный partial-конфиг
    favoriteTypes?: string[];    // избранные типы для быстрого старта
    dailyGoal?: number;          // минут в день
  }
  ```

## 5. Не войдёт в эту версию (Out of Scope)

- ❌ **Адаптивная сложность на основе AI** — авто-подбор сложности по истории ошибок.
- ❌ **Многопользовательские упражнения** — дуэты, ансамбли, teacher-student режим.
- ❌ **Геймификация** — очки, уровни, ачивки, таблицы лидеров.
- ❌ **Загрузка пользовательских упражнений** — создание и публикация своих типов упражнений.
- ❌ **Голосовой ввод** — пение интервалов/мелодий (EarMaster умеет).
- ❌ **Полный курс обучения** — структурированная программа «с нуля до профи».
- ❌ **Экспорт упражнений** — поделиться конфигурацией.
- ❌ **Мобильные приложения** — iOS/Android native.
- ❌ **Визуализация аппликатуры** — гриф гитары, клавиатура фортепиано на карточках.
- ❌ **Авто-транскрипция аудио** — загрузка mp3 → ноты.

## 6. Риски и допущения

| Риск | Решение |
|------|---------|
| Унификация 4+ плагинов под общий ExerciseWizard может сломать существующий UX | Рефакторинг поэтапный: сначала общий компонент, затем миграция плагинов по одному. Обратная совместимость — существующие маршруты (`/ear-training`, `/rhythm-drills`) сохраняются. |
| MIDI-оценка ритма (свинг, полиритмия) требует высокой точности тайминга | Использовать `performance.now()` + квантование с окном допуска. Для свинга — сравнивать отношение длительностей, а не абсолютные значения. |
| Генерация контента для новых типов (диктанты, чтение нот) — нет готовых генераторов | Создать генераторы в `music-core/generator/`: `melodyGenerator`, `rhythmPatternGenerator`, `dictationGenerator`. Переиспользовать существующий `progressionGenerator`. |
| Визуализация нотоносца — новый сложный UI-компонент | Использовать VexFlow (открытая библиотека рендеринга нот) как адаптер к браузеру, изолированный от `music-core`. |
| Multi-chord бары в прогрессиях не поддерживаются генератором | Пока только через ручной DSL-ввод. Расширение генератора — в следующей версии. |
| Пользователь может запутаться в многообразии типов упражнений | Умные дефолты: при первом входе — рекомендация «С чего начать». Быстрый старт с предустановленными параметрами. |

## 7. Метрики успеха

- Пользователь запускает упражнение из любой категории за <60 секунд (от входа в раздел до «▶ Старт»)
- MIDI-оценка работает для всех типов с точностью >95% (ручная верификация на 20 примерах)
- 5 категорий упражнений доступны, минимум 2 подтипа в каждой (MVP)
- Экран результата показывает детализацию ошибок — пользователь понимает, над чем работать
- Единый ExerciseWizard переиспользован во всех типах (нет дублирования кода конфигурации)
- Существующие плагины (`ear-training`, `rhythm-drills`, `chord-quiz`, `practice-cards`) работают без регресса
- `typecheck` + `lint` + `test` — зелёные
- Нотоносец (VexFlow) рендерит ноты без видимых артефактов в Chrome/Firefox/Safari
