# EXERCISE VISION — Интерактивные карточки для тренировки аккордов и мелодий

**Дата:** 2026-06-15
**Горизонт:** 1 версия (3–4 недели)
**Статус:** 🟡 Черновик — ожидает подтверждения

## 1. Резюме (Executive Summary)

Новый раздел тренировки Jazz Trainer: **интерактивные карточки** (`practice-cards`) — визуальная практика джазовых прогрессий и гамм с живым аккомпанементом. Пользователь видит на экране «карточки» с символами аккордов (1, 2 или 3 одновременно — текущий такт, предыдущий и следующий), слышит backing track (бас, барабаны, фортепиано — настраивается), и играет на своём реальном инструменте. Компьютер не оценивает — он показывает и сопровождает.

Ключевая ценность: **самостоятельная практика под аккомпанемент с визуальной подсказкой**, bridging the gap между теорией (справочники аккордов/гамм) и оценкой (MIDI-тесты). Пользователь получает «бесконечный» поток упражнений, генерируемых автоматически, или вводит свои прогрессии через DSL.

**MVP:** 2 типа упражнений — аккордовые прогрессии (P0) и гаммы по прогрессиям (P1). Секвенции и опевания — в следующей версии.

**Отличие от существующих плагинов:**

| Плагин                    | Что делает                  | Оценка?               | Визуальные карточки? | Аккомпанемент?     |
| ------------------------- | --------------------------- | --------------------- | -------------------- | ------------------ |
| `ear-training`            | Распознавание на слух       | ✅ MIDI               | ❌                   | ❌                 |
| `rhythm-drills`           | Повтор ритма                | ✅ MIDI               | ❌                   | ❌                 |
| `chord-quiz`              | Тест на знание аккордов     | ✅ (текст)            | ❌                   | ❌                 |
| `progression-recognition` | Определи прогрессию на слух | ✅ (текст)            | ❌                   | ✅ (аудио)         |
| **`practice-cards`**      | **Играй по карточкам**      | **❌ (не оценивает)** | **✅ 1–3 карточки**  | **✅ полный бэнд** |

## 2. Конкурентный контекст

| Продукт                           | Визуальные карточки               | Авто-генерация прогрессий | Живой аккомпанемент | Джазовая специализация | Браузер      |
| --------------------------------- | --------------------------------- | ------------------------- | ------------------- | ---------------------- | ------------ |
| **iReal Pro**                     | Только chord chart целиком        | ❌ (ручной ввод)          | ✅ MIDI-бэнд        | ✅                     | ❌ (app)     |
| **EarMaster**                     | Ноты на нотоносце                 | ❌                        | ❌                  | ❌                     | ❌ (desktop) |
| **Soundslice**                    | Нотация + синхронизация с YouTube | ❌                        | ❌                  | ❌                     | ✅           |
| **Tenuto / musictheory.net**      | Ноты на нотоносце (статика)       | ❌                        | ❌                  | ❌                     | ✅           |
| **JazzPianoSchool / Open Studio** | Pre-recorded видео                | ❌                        | ❌                  | ❌                     | ✅           |
| **Jazz Trainer (сейчас)**         | ❌                                | ✅ (генератор)            | ✅ (4 инструмента)  | ✅                     | ✅           |
| **Jazz Trainer (цель)**           | ✅ **1–3 карточки с анимацией**   | ✅                        | ✅                  | ✅                     | ✅           |

**Ниша:** Никто не делает **потоковую визуальную практику с авто-генерацией джазовых прогрессий и живым аккомпанементом в браузере**. Это уникальная комбинация: генератор (`music-core/generator`) + аккомпанемент (Bass, Drums, Piano, Rhodes) + карточки (новый UI-слой). Ближайший аналог — iReal Pro, но он показывает всю сетку сразу (нет «телесуфлёра») и требует установки.

## 3. Новые функции (Features)

### 3.1. Мастер настройки упражнения (Exercise Configuration Wizard)

- **Проблема:** Пользователь должен быстро выбрать, что тренировать, и настроить параметры — без чтения документации.
- **Описание:** Трёхшаговый мастер перед стартом упражнения:

  **Шаг 1 — Выбор типа упражнения:**
  Карточки-плитки с иконками: «Аккорды» 🎸, «Гаммы» 🎹, «Секвенции» 🎼 (⏳), «Опевания» 🎵 (⏳). В MVP активны первые два, остальные — с пометкой «Скоро».

  **Индикатор шагов:** при переходе между шагами 1 → 2 → 3 в верхней части экрана отображается stepper (Step 1 · Step 2 · Step 3) с подсветкой текущего шага. На этапе практики stepper скрывается.

  **Шаг 2 — Параметры упражнения (зависят от типа):**

  _Для аккордовых прогрессий:_
  - **Источник:** три варианта на выбор:
    - _Встроенный паттерн_ — выбор из 8 паттернов (`generator/patterns.ts`): ii–V–I major/minor, turnaround, rhythm changes, circle of fifths, dominant chain, modal vamp, random diatonic
    - _Произвольная генерация_ — случайная последовательность диатонических аккордов в выбранной тональности (без привязки к конкретному паттерну)
    - _Ручной ввод DSL_ — текстовое поле с валидацией на лету (`| Dm7 G7 | Cmaj7 |`)
  - **Тональность:** ряд кнопок-чекбоксов с названиями всех 12 тональностей (C, Db, D, Eb, E, F, F#, G, Ab, A, Bb, B). Можно выбрать одну или несколько — тогда прогрессия будет последовательно проигрываться в каждой выбранной тональности.
  - **Повторы:** числовое поле (1–10) + опция «∞ Бесконечно» (воспроизведение не останавливается, пользователь завершает вручную)
  - **Затактов:** 0, 1, 2, 4 такта
  - **Режим карточек:** `Текущий` / `Предыдущий + Текущий` / `Предыдущий + Текущий + Следующий`
  - **Аккомпанемент:** чекбоксы Бас / Барабаны / Фортепиано / Rhodes (можно отключить все — останется только метроном)
  - **Метроном:** вкл/выкл, громкость
  - **Темп:** слайдер 40–300 BPM (значение по умолчанию из настроек пользователя)

  _Для гамм:_
  - **Тип гаммы:** выбор из выпадающего списка: мажор, минор (натуральный, гармонический, мелодический), дорийский, миксолидийский, фригийский, лидийский, локрийский
  - **Контекст:** «Отдельно» (просто гамма от заданной ноты) или «По прогрессии» (для каждого аккорда — соответствующий лад, например: `Dm7 → D дорийский`, `G7 → G миксолидийский`, `Cmaj7 → C ионийский`)
  - **Направление:** вверх / вниз / вверх-вниз
  - **Октав:** 1 или 2
  - Остальные параметры — как для аккордов (тональность, затакты, карточки, темп)

  **Шаг 3 — Инструкция (превью):**
  Показывает что именно будет в упражнении: список аккордов/гамм в текстовом виде + ноты каждого аккорда. Для гамм — запись гаммы нотами на нотоносце (в первой версии — текст, например «D дорийский: D E F G A B C»). Кнопка **«▶ Старт»**.

- **Конкуренты:** Никто не имеет wizard-подхода к настройке; iReal Pro показывает всё сразу (сложно для новичка); EarMaster имеет пошаговый выбор, но без гибкости.
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `practice-cards` (новый плагин, категория `practice`)
- **Зависимости:** 3.2 (Chord Cards), 3.3 (Card Display Engine)

### 3.2. Аккордовые карточки (Chord Progression Cards)

- **Проблема:** Практика компинга по гармонической сетке требует либо смотреть в телефон/ноты (iReal Pro), либо играть на память. Нет инструмента, который ведёт пользователя по прогрессии такт за тактом с живым аккомпанементом.
- **Описание:** Упражнение типа «Аккорды». Генерирует (или парсит из DSL) гармоническую сетку через существующий `generator`. Во время практики:
  1. **Затакт:** метроном отсчитывает заданное число тактов. На экране — крупные точки по центру (как в панели плеера, «светофор» без цветового деления), по одной на каждую долю такта. Точки последовательно подсвечиваются в ритме метронома.
  2. **Карточки:** на экране появляются карточки с символами аккордов. Режим определяется настройкой:
     - `current` — одна карточка по центру, меняется каждый такт.
     - `prev-current` — две карточки: слева предыдущий такт (полупрозрачный), справа текущий (яркий).
     - `prev-current-next` — три карточки: предыдущий (слева, тусклый), текущий (центр, яркий, крупнее), следующий (справа, полупрозрачный).
  3. **Анимация:** карточки смещаются влево с плавной анимацией (slide + fade). Новые появляются справа.
  4. **Аккомпанемент:** BassInstrument + DrumInstrument + PianoInstrument + RhodesInstrument (опционально, согласно настройкам) играют backing track. Если все инструменты отключены — звучит только метроном.
  5. **Цикл:** прогрессия повторяется заданное число раз, затем упражнение завершается (экран с итогами: «Сыграно N тактов в темпе X»).

- **Использует:** `music-core/generator` (8 паттернов + произвольная генерация), `music-core/dsl` (парсинг ручного ввода), `music-core/audio` (TransportEngine + инструменты).
- **Конкуренты:** iReal Pro — только статичный chord chart; **живой «телесуфлёр» с карточками — уникальная фича.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `practice-cards`
- **Зависимости:** 3.1 (Wizard), 3.4 (Card Display Engine). Использует генератор (уже готов).

### 3.3. Гаммы по карточкам (Scale Cards)

- **Проблема:** Практика гамм обычно механическая (вверх-вниз), без музыкального контекста. Джазовая практика требует играть гаммы **в контексте гармонии**: дорийский на минорный аккорд, миксолидийский на доминанту.
- **Описание:** Упражнение типа «Гаммы». Два подрежима:

  **А) Гамма отдельно (standalone):**
  Пользователь выбирает тонику и тип гаммы. Карточки показывают направление («C мажор ↑», «C мажор ↓», «C мажор ↑» — на каждый такт). Пользователь играет гамму вверх-вниз под метроном.

  **Б) Гамма по прогрессии (scale-over-chords):**
  Выбирается гармоническая прогрессия (как в 3.2). Для каждого аккорда автоматически определяется соответствующий лад:
  - `Imaj7` → ионийский
  - `iim7` → дорийский
  - `iiim7` → фригийский
  - `IVmaj7` → лидийский
  - `V7` → миксолидийский
  - `vim7` → эолийский
  - `viim7b5` → локрийский

  Карточка показывает: **символ аккорда + название лада** (например, `Dm7 — D дорийский`). Пользователь играет соответствующую гамму на каждый такт.

- **Конкуренты:** Никто не предлагает практику гамм в гармоническом контексте с аккомпанементом. EarMaster имеет упражнения на гаммы, но без аккомпанемента и визуальных карточек.
- **Приоритет:** P1 (второй тип в MVP)
- **Усиливает плагин / Новая категория:** `practice-cards`
- **Зависимости:** 3.1 (Wizard), 3.2 (Chord Cards — переиспользует логику прогрессий), 3.4 (Card Display Engine).

### 3.4. Движок отображения карточек (Card Display Engine)

- **Проблема:** Нужен переиспользуемый UI-компонент для отображения 1/2/3 карточек с анимацией перехода, работающий синхронно с TransportEngine.
- **Описание:** React-компонент `CardDisplay` внутри плагина `practice-cards`:

  ```ts
  interface CardDisplayProps {
    bars: PracticeBar[]; // все такты упражнения
    currentIndex: number; // индекс текущего такта (из TransportEngine)
    mode: 'current' | 'prev-current' | 'prev-current-next';
  }
  ```

  **Визуальный дизайн карточки:**
  - Фон и общий вид наследуют активную тему приложения (светлая/тёмная), скруглённые углы
  - Крупный шрифт для символов аккордов
  - Текущий такт: яркий, масштаб 1.0, с accent-рамкой (цвет темы)
  - Предыдущий: слева, opacity 0.4, масштаб 0.85
  - Следующий: справа, opacity 0.6, масштаб 0.85
  - Multi-chord бары: символы разделены пробелом (`Dm7 G7`)

  **Анимация:**
  - При смене такта: карточки сдвигаются влево через CSS `transform: translateX()` + `opacity`
  - Длительность: ~300ms (синхронно с долей такта)
  - Использует `requestAnimationFrame` для плавности

  **Интеграция с транспортом:**
  - Подписывается на `TransportEngine` события (позиция в тактах)
  - `currentIndex` обновляется на каждую сильную долю (начало такта)
  - При паузе — карточки замирают (сохраняют позицию)

- **Конкуренты:** Никто не делает анимированный «телесуфлёр» для музыкальной практики.
- **Приоритет:** P0 (MVP — нужен для 3.2 и 3.3)
- **Усиливает плагин / Новая категория:** `practice-cards` (внутренний компонент)
- **Зависимости:** `TransportEngine` (`music-core/audio`).

### 3.5. Интеграция с плеером и аккомпанементом

- **Проблема:** Упражнение с карточками должно управлять воспроизведением (play, pause, stop, темп, стиль) и включать/выключать инструменты аккомпанемента — и делать это через единый, уже знакомый пользователю UX.
- **Описание:**
  - **Единая панель плеера:** в режиме практики используется та же панель управления, что и на странице плеера сетки (`core-player`) — селектор стиля (Swing / Bossa Nova / Funk / Latin / Ballad), кнопки транспорта (Play/Pause/Stop/Loop), слайдер темпа. Пользователь получает знакомый интерфейс без дублирования.
  - **Область карточек:** занимает основную часть экрана вместо стандартной визуализации сетки. Панель плеера располагается над или под карточками (в соответствии с текущим лейаутом `core-player`).
  - **Интеграция с TransportEngine:** плагин создаёт экземпляр `TransportEngine`, загружает в него сгенерированный `GridContent`, подключает выбранные инструменты (Bass, Drums, Piano, Rhodes — через их `InstrumentManifest`). Панель плеера управляет этим экземпляром.
  - **Синхронизация:** `TransportEngine` — единый источник времени и для звука, и для позиции карточек.
  - **Метроном:** визуальный (точки-«светофор» по центру экрана) + звуковой (click track через Tone.js), настраивается независимо от backing track.
  - **Затакт (count-in):** перед первым тактом контента — N тактов только метронома (без аккомпанемента). Визуально: крупные точки по центру экрана, последовательно подсвечиваются в ритме.
  - **Счётчик тактов:** отображается на панели плеера («Такт 3 из 12» или «Такт 5 / ∞»).

- **Конкуренты:** iReal Pro имеет транспорт, но без «телесуфлёра»; EarMaster — без аккомпанемента.
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `practice-cards`, затрагивает `core-player` (возможно извлечение панели плеера как shared-компонента)
- **Зависимости:** 3.4 (Card Display), `music-core/audio` (TransportEngine, Bass/Drum/Piano/Rhodes instruments), `core-player` (панель плеера).

### 3.6. Ручной ввод прогрессий (DSL-режим)

- **Проблема:** Пользователь хочет практиковать конкретную прогрессию (например, из любимого стандарта), а не только встроенные паттерны.
- **Описание:** В шаге 2 мастера, вместо выбора паттерна — текстовое поле для ввода DSL:
  ```
  | Dm7 G7 | Cmaj7 | Fmaj7 | Bm7b5 E7 | Am7 |
  ```
  Валидация на лету через `parseGrid()` из `music-core/dsl`. Ошибки подсвечиваются. Под капотом — вызов `parseGrid()` → `GridContent`, который напрямую подаётся в `TransportEngine`.
- **Конкуренты:** iReal Pro — ручной ввод есть; EarMaster — нет. Но iReal Pro использует свой формат, а не джазовый DSL.
- **Приоритет:** P1 (в составе wizard'а, но может быть отложен если `parseGrid` требует доработок)
- **Усиливает плагин / Новая категория:** `practice-cards`
- **Зависимости:** 3.1 (Wizard), `music-core/dsl`.

## 4. Улучшения существующих функций

### 4.1. Интеграция пользовательских настроек

- Сохранять последние параметры упражнения (тип, тональность, темп, настройки аккомпанемента) в `UserSettingsDTO` через `useSettings()` / `useUpdateSettings()` из `@jazz/plugin-sdk`.
- Новые поля в `UserSettingsDTO` (предложение):
  ```ts
  practiceCards: {
    lastExerciseType?: 'chords' | 'scales';
    lastSource?: 'pattern' | 'random' | 'dsl';
    lastPatternId?: string;
    lastKeys?: Key[];
    lastTempo?: number;
    lastRepetitions?: number;
    lastInfinite?: boolean;
    cardMode?: 'current' | 'prev-current' | 'prev-current-next';
    countInBars?: number;
    backingBass?: boolean;
    backingDrums?: boolean;
    backingPiano?: boolean;
    backingRhodes?: boolean;
    metronomeEnabled?: boolean;
    metronomeVolume?: number;
  }
  ```

### 4.2. Экран завершения упражнения

- После окончания упражнения (все повторы пройдены или пользователь нажал Stop в бесконечном режиме) — экран с краткой сводкой: «12 тактов в темпе 120 BPM, тональность C, паттерн ii-V-I major».
- Кнопки: «Повторить» (те же настройки), «Настроить заново» (шаг 2 мастера), «Закончить» (на главную).

## 5. Не войдёт в эту версию (Out of Scope)

- ❌ **MIDI-отслеживание и оценка игры пользователя** — по явному требованию, отложено до Ф5/следующей версии.
- ❌ **Секвенции и опевания** — вторые 2 типа упражнений из оригинальной идеи. Будут добавлены после MVP, когда отработает движок карточек.
- ❌ **Визуализация аппликатуры** (клавиатура / гриф) — отложено.
- ❌ **Ноты на нотоносце** на карточках — пока только символы аккордов и названия гамм. Нотоносец — в следующей версии.
- ❌ **Редактор пользовательских паттернов** — создание и сохранение своих гармонических шаблонов.
- ❌ **Exercise history и streaks** — сохранение истории упражнений, подсчёт «дней подряд».
- ❌ **Sharing / экспорт упражнений** — поделиться настройками упражнения с другим пользователем.
- ❌ **Авто-подбор сложности** — адаптивное усложнение на основе истории.
- ❌ **Полноэкранный режим** с крупными карточками для игры с расстояния.

| Синхронизация анимации карточек со звуком может «плыть» (requestAnimationFrame ≠ аудио-буфер)  | Использовать события `TransportEngine` (bar/beat) как источник истины; анимация — чисто визуальная, допускает небольшую погрешность. CSS transition 300ms маскирует микро-расхождения.                                                                                                                     |
| Генератор даёт один аккорд на такт, но в джазе часты multi-chord бары (`Dm7 G7` в одном такте) | Генератор поддерживает только одно-аккордные такты. Multi-chord — только через ручной DSL-ввод (`3.6`). В будущем — расширение генератора на multi-chord.                                                                                                                                                  |
| Пользователь может запутаться в настройках (wizard из 3 шагов может показаться длинным)        | Шаг 2 можно сократить: «быстрый старт» — выбор паттерна + тональности, всё остальное по умолчанию. Кнопка «▶ Быстрый старт» сразу запускает упражнение с дефолтными параметрами.                                                                                                                           |
| Мобильные устройства — карточки могут быть мелкими (особенно 3 одновременно)                   | На мобильных (<768px) режим `prev-current-next` форсируется в `current` (одна карточка). Адаптивная вёрстка.                                                                                                                                                                                               |
| Конфликт с существующим плеером (`core-player`) — два источника управления транспортом         | `practice-cards` создаёт собственный экземпляр `TransportEngine` для упражнения, но повторно использует **панель плеера** (style selector, transport, tempo) из `core-player` как shared-компонент. Упражнение и плеер сетки не пересекаются по данным — это разные экземпляры транспорта с одинаковым UX. |

## 7. Метрики успеха

- Пользователь проходит wizard за <60 секунд и запускает упражнение
- Карточки сменяются синхронно с началом такта (визуальная проверка)
- Аккомпанемент (бас + барабаны + фортепиано + Rhodes) играет в выбранном стиле и темпе; при отключении всех инструментов — только метроном
- Счётчик тактов корректен, в бесконечном режиме показывает «Такт N / ∞»
- Переключение режима карточек (1/2/3) работает без перезагрузки упражнения
- Ручной DSL-ввод парсится без ошибок, упражнение запускается
- Выбор нескольких тональностей — прогрессия последовательно проходит по всем выбранным
- Панель плеера (`core-player`) переиспользована, стиль меняется глобально
- `typecheck` + `lint` + `test` — зелёные
- Существующие тесты `music-core/generator` и `music-core/dsl` не затронуты

## Приложение В: Модель данных карточки

```ts
interface PracticeBar {
  /** Индекс такта (0-based) */
  index: number;
  /** Символы аккордов в такте, например ["Dm7"] или ["Dm7", "G7"] */
  chords: string[];
  /** Метка лада для гамм (опционально) */
  scaleLabel?: string;
}

interface ExerciseSession {
  type: 'chords' | 'scales';
  bars: PracticeBar[];
  config: ExerciseConfig;
}
```
