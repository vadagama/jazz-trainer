# CATALOG VISION — Публичный каталог композиций

**Дата:** 2026-07-14
**Статус:** 🟡 Черновик — ожидает подтверждения
**Версия:** v1.0

## 1. Резюме (Executive Summary)

Сейчас каталог Jazz Trainer (`/`) показывает все публичные гармонические композиции единым списком с базовым поиском по названию, стилю и автору. Это работает, но не решает задачи **обнаружения** (discovery) контента: пользователь не может быстро найти «джазовый стандарт в размере 3/4 уровня beginner» или понять по карточке, что внутри композиции, не открывая её.

**Цель:** превратить каталог из «списка всего подряд» в **структурированную витрину** композиций (jazz standards, авторские произведения) с богатыми метаданными, фильтрами и превью (текстовым и аудио).

**Модель наполнения:** каталог — **курируемый**, а не user-generated. Публикуют композиции только пользователи с ролью **Catalog Editor** (редактор каталога). Обычные пользователи — только потребляют (просмотр, лайки, прослушивание). Это гарантирует качество контента и отсутствие модерационного хаоса на старте.

**Ключевая ценность:**

- **Для ученика:** «Хочу практиковать блюз в F, уровень beginner» → поиск по стилю и уровню → открыть в плеере.
- **Для преподавателя:** «Нужен стандарт All The Things You Are для разбора на уроке» → поиск по автору (Jerome Kern) → превью → открыть в плеере.
- **Для редактора каталога:** «Добавил композицию Autumn Leaves» → заполнил метаданные → опубликовал → доступна всем.

**MVP:** расширение метаданных композиций, фильтры, текстовое превью. Аудио-превью — P1 (следующая итерация).

## 2. Структура сущностей каталога

### 2.1. Единая сущность: Композиция

Каталог содержит один тип сущности — **Композиция** — законченное музыкальное произведение: джазовый стандарт, авторская пьеса, этюд. Каждая композиция основана на гармонической структуре (`HarmonyComposition`), что позволяет использовать общий плеер, редактор и DSL.

Композиция имеет: автора-композитора, стиль, размер, темп, уровень сложности, теги.

### 2.2. Метаданные карточки каталога

#### Поля каталога

| Поле               | Тип    | Обязательное | Описание                                                                                   |
| ------------------ | ------ | ------------ | ------------------------------------------------------------------------------------------ |
| `author`           | string | ✅           | Оригинальный композитор/автор: «Oscar Peterson», «Duke Ellington», «Antonio Carlos Jobim»  |
| `recommendedStyle` | Style  | ✅           | Рекомендованный стиль: `'swing'` \| `'bossa'` \| `'funk'` \| `'latin'` \| `'ballad'`       |
| `recommendedTempo` | number | ✅           | Рекомендованный BPM (целое, 20–400)                                                        |

> **Примечание:** композиция попадает в публичный каталог только через редактора каталога (`catalog_editor`). Обычный пользователь не может опубликовать свою композицию в каталог — его композиции остаются в личном разделе «Мои композиции».

### 2.3. Теги — контролируемый словарь

Теги не свободные (не `#чтоугодно`) — используется **закрытый словарь**, расширяемый администратором:

| Категория тегов     | Примеры значений                                                                                         |
| ------------------- | -------------------------------------------------------------------------------------------------------- |
| **Жанр / Форма**    | `jazz-standard`, `blues`, `bebop`, `modal`, `latin`, `funk`, `ballad`, `free-jazz`                       |
| **Гармония**        | `ii-V-I`, `rhythm-changes`, `coltrane-changes`, `modal-interchange`, `turnaround`, `tritone-sub`, `secondary-dominants`, `diminished` |
| **Тип ансамбля**    | `trio`, `quartet`, `quintet`, `big-band`, `solo-piano`                                                   |
| **Методика**        | `voice-leading`, `rootless-voicings`, `shell-voicings`, `walking-bass`, `comping`                        |

**Почему закрытый словарь:** свободные теги приводят к дубликатам (`blues` vs `блюз` vs `Blues`), орфографическим ошибкам и невозможности построить нормальную фильтрацию. Администратор управляет словарём через админку.

### 2.5. Расширение схемы БД

К существующей таблице `harmony_grids` (будет переименована в `harmony_compositions`, см. §12) добавляются колонки:

```sql
ALTER TABLE harmony_compositions ADD COLUMN description TEXT;
ALTER TABLE harmony_compositions ADD COLUMN difficulty TEXT NOT NULL DEFAULT 'intermediate'
  CHECK (difficulty IN ('beginner', 'intermediate', 'advanced'));

ALTER TABLE harmony_compositions ADD COLUMN tags TEXT NOT NULL DEFAULT '[]';  -- JSON-массив строк

ALTER TABLE harmony_compositions ADD COLUMN author TEXT NOT NULL DEFAULT '';
ALTER TABLE harmony_compositions ADD COLUMN recommended_style TEXT;  -- Style, NULL = 'swing'
ALTER TABLE harmony_compositions ADD COLUMN recommended_tempo INTEGER;  -- BPM

ALTER TABLE harmony_compositions ADD COLUMN catalog_published_at INTEGER NOT NULL DEFAULT (unixepoch());
ALTER TABLE harmony_compositions ADD COLUMN copy_count INTEGER NOT NULL DEFAULT 0;

ALTER TABLE harmony_compositions ADD COLUMN featured INTEGER NOT NULL DEFAULT 0;  -- boolean 0/1
ALTER TABLE harmony_compositions ADD COLUMN featured_order INTEGER;

ALTER TABLE harmony_compositions ADD COLUMN moderation_status TEXT NOT NULL DEFAULT 'approved'
  CHECK (moderation_status IN ('approved', 'rejected'));
```

Добавляются индексы для поиска:

```sql
CREATE INDEX idx_compositions_author ON harmony_compositions(author);
CREATE INDEX idx_compositions_difficulty ON harmony_compositions(difficulty);
CREATE INDEX idx_compositions_recommended_style ON harmony_compositions(recommended_style);
CREATE INDEX idx_compositions_featured ON harmony_compositions(featured);
CREATE INDEX idx_compositions_moderation_status ON harmony_compositions(moderation_status);
CREATE INDEX idx_compositions_catalog_published_at ON harmony_compositions(catalog_published_at);
```

## 3. Фильтры и поиск

### 3.1. Быстрые фильтры (UI)

На странице каталога — панель фильтров над списком карточек:

| Фильтр              | Тип               | Значения                                                                                  |
| ------------------- | ----------------- | ----------------------------------------------------------------------------------------- |
| **Стиль**           | multi-select      | Swing · Bossa · Funk · Latin · Ballad                                                     |
| **Размер**          | multi-select      | 2/4 · 3/4 · 4/4 · 5/4 · 6/8 · 7/4 · 12/8                                                 |
| **Уровень**         | multi-select      | Beginner · Intermediate · Advanced                                                        |
| **Тональность**     | multi-select      | C · Db · D · Eb · E · F · F# · G · Ab · A · Bb · B (группировка по бемолям/диезам)       |
| **Теги**            | chips / dropdown  | jazz-standard · blues · bebop · modal · ii-V-I · …                                        |
| **Автор**           | autocomplete      | Поиск по полю `author` (композитор)                                                       |
| **Публикатор**      | autocomplete      | Поиск по полю `publisherName`                                                             |
| **Темп (BPM)**      | range slider      | От 20 до 400                                                                              |
| **Количество тактов**| range slider     | От 1 до N (макс. значение из БД)                                                          |

### 3.2. Полнотекстовый поиск

Единая поисковая строка (над фильтрами) ищет **одновременно** по:

- `name` — название композиции
- `author` — оригинальный композитор
- `description` — текстовое описание
- `tags` — теги

**Механика:** SQLite `LIKE '%query%'` для MVP (индексы на `name`, `author`). При масштабировании — переход на FTS5 (полнотекстовый индекс SQLite, уже встроен в SQLite, не требует внешних зависимостей).

**Примеры поисковых запросов:**

- `"autumn"` → находит «Autumn Leaves», «Autumn Serenade»
- `"jobim"` → все композиции Antonio Carlos Jobim
- `"blues Bb"` → блюзы в тональности Bb

### 3.3. Сортировка

| Критерий     | Описание                                                       |
| ------------ | -------------------------------------------------------------- |
| `popular`    | По убыванию лайков (по умолчанию)                              |
| `newest`     | По дате публикации в каталог (`catalogPublishedAt`)            |
| `updated`    | По дате последнего редактирования                              |
| `name_asc`   | По алфавиту (A→Z)                                              |
| `copies`     | По количеству копирований                                      |
| `featured`   | Избранное администратором — всегда в топе, затем по `popular`  |

### 3.4. URL-параметры (шарябельные ссылки)

Все фильтры и поиск отражаются в URL — пользователь может скопировать ссылку и поделиться:

```
/catalog?style=swing&difficulty=beginner&tags=blues,ii-V-I&q=autumn&sort=popular
```

При переходе по ссылке фильтры применяются автоматически.

## 4. Превью карточки: текст и аудио

### 4.1. Текстовое превью

В карточке каталога показываются **первые 4 такта** композиции в сокращённом DSL-формате:

```
| Fm7 | Bbm7 | Eb7 | Abmaj7 | ...
```

**Реализация:** берём первые 4 элемента из `Content.bars`, форматируем в DSL-строку. Не требует дополнительных вычислений — читаем из `content` поля записи.

### 4.2. Аудио-превью (P1 — вторая итерация)

Каждая карточка в каталоге имеет кнопку ▶ **Прослушать** — короткий (15–30 сек) аудио-фрагмент.

**Как генерируется:**

1. При публикации композиции в каталог (или по запросу администратора «сгенерировать превью») сервер запускает **офлайн-рендеринг** первых N тактов через `TransportEngine`.
2. Результат — WAV/MP3-файл, сохраняется в файловое хранилище (папка `data/previews/`).
3. В карточке — HTML5 `<audio>` с кнопкой play/pause.

**Технические детали:**

- Рендеринг через Tone.js в Node.js (используя `OfflineAudioContext`).
- Длительность превью: 15 секунд или 2 повторения композиции (что короче).
- Инструменты: только Piano + Bass + Drums (без Rhodes/Guitar — для скорости рендеринга).
- Формат: MP3 128 kbps (малый размер для быстрой загрузки).
- Поле `preview_url` в таблице `harmony_compositions` (NULL, пока не сгенерировано).

**Важно:** аудио-превью — P1, не блокирует MVP каталога. Текстовое превью работает сразу.

## 5. Администрирование каталога

### 5.1. Плагин администрирования каталога

**Новый плагин:** `admin-catalog` | **Маршрут:** `/admin/catalog` | **Категория:** `admin`

Входит в существующую экосистему админ-плагинов (7 шт.), использует те же permissions и audit.

### 5.2. Публикация и модерация

**Кто публикует:** только пользователи с ролью `catalog_editor` (или `admin`/`super_admin`). Обычные пользователи (`user`) **не могут** публиковать в каталог.

**Процесс публикации (редактор каталога):**

1. Редактор переходит в `/admin/catalog/publish`
2. Либо создаёт новую композицию с нуля, либо выбирает существующую из своих «Моих композиций»
3. Заполняет метаданные: `name`, `author`, `description`, `difficulty`, `tags`, `recommendedStyle`, `recommendedTempo`
4. Нажимает «Опубликовать» — композиция появляется в каталоге сразу (`moderation_status = 'approved'`)

**Модерация (admin+):** администратор может скрыть/отклонить любую публикацию, если она не соответствует стандартам качества.

#### Таблица модерации

```
┌────────────┬────────────────────┬──────────┬──────────┬──────────┐
│ Название   │ Автор / Публикатор  │ Дата публ│ Статус   │ Действия │
├────────────┼────────────────────┼──────────┼──────────┼──────────┤
│ Autumn…    │ J. Kern / I.Petrov │ 12.07.26 │ ✅ Одобр.│ ✏️ 👁️ 🗑️ │
│ Giant Steps│ J.Coltrane / A.Jazz│ 10.07.26 │ 🚫 Откл. │ ↩️       │
└────────────┴────────────────────┴──────────┴──────────┴──────────┘
```

**Действия модератора:**

- 👁️ **Просмотреть** — открыть композицию в плеере для оценки содержания
- 🗑️ **Скрыть** — установить `moderation_status = 'rejected'` (композиция исчезает из каталога)
- ↩️ **Восстановить** — вернуть `moderation_status = 'approved'`
- ✏️ **Редактировать** — изменить метаданные (теги, сложность, etc.)

### 5.3. Избранное (Featured)

Администратор может отметить до 10 композиций как «избранные» — они показываются в специальном блоке **вверху каталога** (карусель / горизонтальный скролл).

**Управление:**

- В таблице модерации — звёздочка ⭐ для добавления/удаления из избранного.
- Drag-and-drop для изменения порядка (`featuredOrder`).
- Избранные композиции помечаются бейджем «⭐ Выбор редакции» в общем списке.

### 5.4. Управление тегами

Новый раздел в плагине: `/admin/catalog/tags`.

- Таблица всех тегов с количеством использований.
- **Добавить тег:** название, категория (из словаря §2.3), описание.
- **Переименовать тег:** каскадное обновление во всех композициях.
- **Удалить тег:** с подтверждением (тег удаляется из всех композиций).
- **Объединить теги:** «объединить `blues` и `блюз` → `blues`» (каскадная замена).
- **Скрыть тег:** тег не показывается в фильтрах, но остаётся на существующих композициях.

### 5.5. Массовые операции

- Выделить несколько композиций (checkboxes) → массово:
  - Скрыть / Восстановить
  - Добавить / убрать тег(и)
  - Изменить уровень сложности
  - Удалить

### 5.6. Статистика каталога

Дашборд в админке (`/admin/catalog/stats`):

| Метрика                     | Описание                                          |
| --------------------------- | ------------------------------------------------- |
| Всего композиций            | `COUNT(*)`                                         |
| Опубликовано за неделю      | График по дням                                    |
| Топ-10 по лайкам            | Таблица                                           |
| Топ-10 по копированиям      | Таблица                                           |
| Распределение по стилям     | Pie chart (swing / bossa / funk / latin / ballad) |
| Распределение по сложности  | Bar chart (beginner / intermediate / advanced)    |

### 5.7. Права доступа

**Новая роль:** `catalog_editor` — редактор каталога. Может создавать, редактировать и публиковать композиции в каталог. Не имеет административных прав (модерация, featured, теги — только у `admin`/`super_admin`).

Расширение существующей RBAC-модели:

| Permission              | Описание                                        | Кто имеет                                            |
| ----------------------- | ----------------------------------------------- | ---------------------------------------------------- |
| `catalog:read`          | Просмотр каталога (публичный)                   | Все (включая анонимов)                               |
| `catalog:publish`       | Создание и публикация композиций в каталог      | `catalog_editor`, `admin`, `super_admin`              |
| `catalog:moderate`      | Модерация: скрыть/восстановить/удалить          | `admin`, `super_admin`                               |
| `catalog:feature`       | Управление избранным (featured)                 | `admin`, `super_admin`                               |
| `catalog:tags:write`    | Управление тегами                               | `admin`, `super_admin`                               |
| `catalog:stats:read`    | Просмотр статистики каталога                    | `admin`, `super_admin`                               |

**Обычный пользователь (`user`)** не имеет права публиковать в каталог. Его роль в каталоге — только потребление: просмотр, поиск, лайки, прослушивание.

## 6. API-эндпоинты

### 6.1. Публичные (без авторизации)

| Метод  | Путь                         | Описание                                                          |
| ------ | ---------------------------- | ----------------------------------------------------------------- |
| `GET`  | `/api/catalog`               | Список публикаций с фильтрацией, поиском, сортировкой и пагинацией |
| `GET`  | `/api/catalog/:id`           | Детали публикации (поля + превью аккордов)                        |

| Метод   | Путь                            | Описание                                    |

## 8. Технические аспекты

### 8.1. Миграции БД

- Все новые колонки — с дефолтными значениями (`NOT NULL DEFAULT ...`), обратно совместимы.
- Существующие публичные композиции автоматически получают `difficulty = 'intermediate'`, `moderation_status = 'approved'`.
- Поле `catalog_published_at` для существующих композиций = `created_at`.
- Миграция через Drizzle Kit (уже используется в проекте).

### 8.2. Обновление Zod-DTO

В `packages/shared/src/compositions.ts` (переименован из `grids.ts`, см. §12) добавляются расширенные схемы для каталога:

- `CatalogEntrySchema` — полные метаданные
- `CatalogQuerySchema` — query-параметры фильтрации
- `CreateCatalogEntrySchema` — валидация при публикации
- `UpdateCatalogEntrySchema` — валидация при редактировании

### 8.3. Обновление PublicCompositionSummaryDTO

Существующая схема `PublicGridSummaryDTO` (будет переименована в `PublicCompositionSummaryDTO`) расширяется новыми полями каталога (обратная совместимость через `extend`).

### 8.4. Аудит

Все административные действия логируются через существующий `withAudit()`:

```ts
await withAudit({
  actorId: req.user.id,
  action: 'catalog:reject',
  entityType: 'catalog_entry',
  entityId: compositionId,
  newValues: { moderation_status: 'rejected' },
});
```

## 10. Риски и допущения

| Риск                                          | Вероятность | Смягчение                                                             |
| --------------------------------------------- | ----------- | --------------------------------------------------------------------- |
| Расширение `harmony_compositions` сломает обратную совместимость | Низкая      | Все новые колонки `NOT NULL DEFAULT`, старые API не затрагиваются     |
| Закрытый словарь тегов покажется ограниченным | Средняя     | Администратор может расширять словарь в любой момент                  |
| Большое количество фильтров усложнит UX       | Средняя     | Дизайн-ревью перед реализацией, user-testing                          |
| Аудио-превью потребует много CPU              | Средняя     | Генерация в фоне (очередь), кэширование, только 15 секунд             |
| Рост размера БД из-за новых индексов          | Низкая      | SQLite хорошо справляется с индексами на десятках тысяч записей       |
| Роль `catalog_editor` усложнит RBAC           | Низкая      | Добавление одной роли + 1 permission — минимальное расширение         |

## 12. Переименование «Сетка → Композиция»

**Предварительное условие** для запуска каталога: глобальная замена терминологии по всему проекту.

### 12.1. Что меняется

| Было | Стало |
|------|-------|
| Гармоническая сетка | Композиция |
| Мои сетки | Мои композиции |
| `HarmonyGrid` (в коде) | `HarmonyComposition` |
| `harmony_grids` (таблица БД) | `harmony_compositions` |
| `/api/grids` (API-путь) | `/api/compositions` |
| `grids.ts` (файлы) | `compositions.ts` |
| `MyGridsPage.tsx` | `MyCompositionsPage.tsx` |
| `MyGridCard.tsx` | `MyCompositionCard.tsx` |
| `CreateGridDialog.tsx` | `CreateCompositionDialog.tsx` |

### 12.2. Объём изменений

Затрагивает ~15+ файлов в трёх слоях:

| Слой | Файлы | Характер изменений |
|------|-------|--------------------|
| `packages/shared/` | `grids.ts` → `compositions.ts`, `music.ts`, `index.ts` | Типы, Zod-схемы, переименование файла |
| `apps/api/` | `schema.ts`, `*.routes.ts`, `*.service.ts` | Таблица БД, эндпоинты, миграция |
| `apps/web/` + плагины | `Header.tsx`, `DashboardPage.tsx`, `MyGridsPage.tsx` → `MyCompositionsPage.tsx`, `MyGridCard.tsx` → `MyCompositionCard.tsx`, `CreateGridDialog.tsx` → `CreateCompositionDialog.tsx`, `EditorPage.tsx`, `CatalogPage.tsx`, `PublicGridCard.tsx` → `PublicCompositionCard.tsx`, `LoginPage.tsx` | UI-тексты, маршруты, имена компонентов |

### 12.3. Порядок выполнения

1. `packages/shared/` — типы и DTO (основа для всех остальных)
2. `apps/api/` — БД и эндпоинты
3. Плагины и `apps/web/` — UI и маршруты
4. Документация (`CLAUDE.md`, `README.md`, `docs/*.md`) и алиасы (`tsconfig.base.json`, `vite.config.ts`, `vitest.config.ts`)

**Сложность:** M (3–5 дней) — механическая замена, но требует аккуратности (имена файлов, импорты, алиасы). Без этого переименования запускать каталог с термином «сетка» будет терминологически некорректно.
