## 1. Резюме (Executive Summary)

**Секвенции** (melodic sequences) — четвёртый тип упражнений в плагине `practice-cards`, изначально запланированный в `EXERSISE-VISION.md` §5 как «скоро». Плитка «Секвенции» уже есть в `StepTypeSelect`, но сейчас `disabled: true`. Задача этой итерации — активировать плитку и реализовать полноценное упражнение.

По замыслу пользователя **упражнения по секвенциям делаются по образцу гамм и опеваний**:

- Тот же discriminated union `ExerciseConfig`.
- Та же базовая конфигурация `BaseExerciseConfig` (тональности, повторы, карточки, аккомпанемент, метроном).
- Те же источники гармонии (`ChordSource`): `unified` (отдельно), `pattern`, `random`, `dsl`.
- Те же хелперы генерации из `core.ts` (`extractChordsFromSource`, `toPracticeBars`, `repeatBars`, `expandBarsPerChord`, `shuffle`).
- Те же режимы отображения карточек (`current` / `prev-current` / `prev-current-next`).

Отличие от гамм — вместо простого «лада вверх/вниз» на карточке показывается **мелодический паттерн, транспонированный на очередную ступень лада**. Пользователь тренирует «мелодический словарь» — стандартные джазовые ячейки (1235, 1357 и т.д.), которые повторяются с разных ступеней и формируют беглость на инструменте.

**Ключевая ценность:** пользователь видит аккорд + стартовую ступень + ноты паттерна, слышит живой аккомпанемент и отрабатывает бибоп/джазовый мелодический словарь в разных регистрах. Это мост между «знаю гамму» и «умею строить мелодию из гаммы» — в формате потока карточек, который не предлагает ни один конкурент.

**MVP:** 11 базовых паттернов в 4 категориях (восходящие, нисходящие, терцовые/skip-step, повороты/арпеджио), опция «все (случайно)», стартовые ступени 1–7, направление вверх/вниз/оба, два режима — «отдельно» (`unified`) и «по прогрессии» (`pattern` / `random` / `dsl`).

## 2. Что такое секвенция (музыкальная справка)

**Секвенция (sequence)** — мелодический приём, при котором короткий мелодический паттерн повторяется, начиная с разных ступеней лада. Каждое повторение («звено секвенции») транспонировано диатонически — то есть сохраняет форму интервального паттерна, но «скользит» по ладу.

В джазе и бибопе это базовый словарный материал: паттерны `1-2-3-5`, `1-3-5-7`, `5-3-2-1`, `3-5-7-9`, пентатонные и терцовые ходы формируют «кирпичики», из которых строится импровизация. Их отрабатывают во всех тональностях и с разных ступеней, чтобы палец и ухо знали форму независимо от стартовой ноты.

### 2.1. Базовые паттерны (расширенный MVP)

Все паттерны — чистые диатонические последовательности ступеней лада (без хроматики). Каждый паттерн транспонируется на стартовую ступень и повторяется на следующем звене секвенции.

#### Восходящие (Ascending)

| Тип | Формула (ступени) | Пример в C мажоре (от I) | Название в UI |
| --- | --- | --- | --- |
| **1235** | 1-2-3-5 | `C-D-E-G` | 1-2-3-5 (бибоп) |
| **1234** | 1-2-3-4 | `C-D-E-F` | 1-2-3-4 |
| **1357** | 1-3-5-7 | `C-E-G-B` | 1-3-5-7 (арпеджио) |
| **1531** | 1-5-3-1 | `C-G-E-C` | 1-5-3-1 (период) |
| **pentatonic** | 1-2-3-5-6 | `C-D-E-G-A` | Пентатоника |

#### Нисходящие (Descending)

| Тип | Формула (ступени) | Пример в C мажоре (от I) | Название в UI |
| --- | --- | --- | --- |
| **5321** | 5-3-2-1 | `G-E-D-C` | 5-3-2-1 (нисх. бибоп) |
| **8765** | 8-7-6-5 | `C-B-A-G` | 8-7-6-5 (нисх. тетрахорд) |

#### Терцовые и skip-step

| Тип | Формула (ступени) | Пример в C мажоре (от I) | Название в UI |
| --- | --- | --- | --- |
| **1324** | 1-3-2-4 | `C-E-D-F` | 1-3-2-4 (терции) |
| **1345** | 1-3-4-5 | `C-E-F-G` | 1-3-4-5 (skip-step) |
| **1356** | 1-3-5-6 | `C-E-G-A` | 1-3-5-6 (maj6) |

#### Повороты и расширенные арпеджио

| Тип | Формула (ступени) | Пример в C мажоре (от I) | Название в UI |
| --- | --- | --- | --- |
| **1231** | 1-2-3-1 | `C-D-E-C` | 1-2-3-1 (поворот) |
| **3212** | 3-2-1-2 | `E-D-C-D` | 3-2-1-2 (нижн. поворот) |
| **3579** | 3-5-7-9 | `E-G-B-D` | 3-5-7-9 (арпеджио 9) |

> В UI добавляется опция **«Все (случайно)»** — на каждый такт случайно выбирается один из 11 паттернов. Паттерны группируются по категориям для удобства выбора.

### 2.2. Стартовые ступени и направление

Секвенция поднимается или опускается по ладу — паттерн транспонируется на очередную ступень:

| Параметр | Значения | Описание |
| --- | --- | --- |
| **Стартовые ступени** | `1, 2, 3, 4, 5, 6, 7` | С каких ступеней лада стартует паттерн. По умолчанию `1-5` (5 звеньев). |
| **Направление** | `up` / `down` / `both` | Порядок обхода ступеней. `both` = вверх, затем вниз. |

Пользователь может выбрать одну или несколько стартовых ступеней. Если выбрано несколько — в режиме `unified` генерируется такт на каждую выбранную ступень. Если выбрана одна — отработка одного звена.

### 2.3. Почему секвенции похожи на гаммы

| Аспект | Гаммы | Секвенции |
| --- | --- | --- |
| Режим «отдельно» | `source.type === 'unified'` | `source.type === 'unified'` |
| Режим «по прогрессии» | `source.type === 'pattern' / 'random' / 'dsl'` | `source.type === 'pattern' / 'random' / 'dsl'` |
| Карточка | аккорд + лад | аккорд + стартовая ступень + ноты паттерна |
| Что играет пользователь | лад вверх/вниз | паттерн, транспонированный на ступень |
| Направление | `up` / `down` / `both` | `up` / `down` / `both` |
| Генератор | `scaleExercise.ts` | `sequenceExercise.ts` |
| UI-шаг | `StepScaleConfig.tsx` | `StepSequenceConfig.tsx` |

## 3. Новые функции (Features)

### 3.1. Активация плитки «Секвенции»

- **Проблема:** плитка «Секвенции» существует, но `disabled: true`, а тип `onSelect` не включает `'sequences'`.
- **Описание:** снять `disabled` с плитки, заменить иконку `⏳` → `🎯`, убрать «(скоро)» из описания, расширить типы `onSelect` и `ExerciseKind` до `'chords' | 'scales' | 'enclosures' | 'sequences'`.
- **Приоритет:** P0
- **Зависимости:** —

### 3.2. Мастер настройки секвенций

- **Проблема:** пользователь должен выбрать тип паттерна, стартовые ступени, направление и источник гармонии.
- **Описание:** новый компонент `StepSequenceConfig.tsx` (по образцу `StepScaleConfig.tsx` / `StepEnclosureConfig.tsx`):
  - **Тип секвенции:** выпадающий список из 11 паттернов (сгруппированы по категориям: восходящие, нисходящие, терцовые/skip-step, повороты/арпеджио) + «Все (случайно)».
  - **Стартовые ступени:** чекбоксы `1-7` (множественный выбор).
  - **Направление:** `up` / `down` / `both`.
  - **Лад для секвенций:** `Select` из 9 ладов (через `SCALE_TYPES` / `SCALE_LABELS`).
  - **Источник:** `SourceCard` с теми же вкладками, что и у гамм (`unified`, `pattern`, `random`, `dsl`).
  - Остальные параметры — общие карточки: тональности, повторы, бесконечность, затакт, режим карточек, аккомпанемент, метроном, темп, рандомизация, тактов на элемент.
- **Приоритет:** P0
- **Зависимости:** 3.1

### 3.3. Генератор упражнений по секвенциям

- **Проблема:** нужен чистый модуль `Config → PracticeBar[]`, аналогичный `scaleExercise.ts`.
- **Описание:** реализовать `generateSequenceExercise(config, rng)`:
  - При `source.type === 'unified'` — строить тонический аккорд для каждой выбранной тональности и генерировать такты для каждой выбранной стартовой ступени.
  - При `source.type !== 'unified'` — получить прогрессию через `extractChordsFromSource`, на каждый аккорд назначить стартовую ступень (циклически из выбранных) и построить паттерн от аккордового тона.
  - Поддержать `playRandomly`, `barsPerChord`, `repetitions`, `infinite`, `direction` — теми же утилитами, что и гаммы/опевания.
- **Приоритет:** P0
- **Зависимости:** 3.4 (доменный модуль `sequences.ts`)

### 3.4. Доменный модуль секвенций в `music-core`

- **Проблема:** расчёт нот паттерна, транспонированного на ступень лада, — чистая музыкальная теория, которая пригодится и для будущей MIDI-оценки, и для лекций.
- **Описание:** создать `packages/music-core/src/chords/sequences.ts`:
  - Типы: `SequenceType`, `ConcreteSequenceType`, `SequenceNote`, `SequenceNoteRole`.
  - Каталог: `SEQUENCE_PATTERNS` (индексы ступеней лада для каждого паттерна), `CONCRETE_SEQUENCE_TYPES`.
  - Функции: `resolveSequencePattern(startPc, type, key, scaleType)`, `buildSequenceCycle(...)`, `randomSequenceType(rng)`.
  - Реэкспорт через `music-core/src/chords/index.ts`.
- **Приоритет:** P0
- **Зависимости:** `SCALE_INTERVALS`, `scalePitchClasses` (из `enclosures.ts`), `keyToPitchClass`, `spellPitchClass`.

### 3.5. Отображение секвенций в карточках

- **Проблема:** `PracticeBar` и `CardDisplay` не умеют показывать стартовую ступень и ноты паттерна.
- **Описание:**
  - Расширить `PracticeBar` опциональным полем `sequence?: { type; startDegree; notes; direction }`.
  - В `CardDisplay` добавить ветку отображения секвенции:
    - строка 1: `Ступень N`;
    - строка 2: крупный символ аккорда;
    - строка 3: ноты паттерна через пробел, корневая нота выделена accent-цветом.
- **Приоритет:** P0
- **Зависимости:** 3.3

### 3.6. Сохранение настроек

- **Проблема:** последние параметры секвенций должны сохраняться между сессиями.
- **Описание:** расширить `UserSettingsDTO.practiceCards` полями `lastExerciseType` (добавить `'sequences'`), `lastSequenceType`, `lastSequenceStartDegrees`, `lastSequenceScaleType`. Читать и сохранять их в `ExerciseWizard`.
- **Приоритет:** P1
- **Зависимости:** 3.2

## 5. Не войдёт в эту версию (Out of Scope)

- ❌ MIDI-оценка секвенций — отложено до Ф5.
- ❌ Пользовательские паттерны (своя формула ступеней).
- ❌ Хроматические секвенции (строго по полутонам, не по ладу).
- ❌ Ритмические паттерны (синкопы, восьмые/триоли) — секвенция показывается как набор нот, ритм выбирает пользователь.
- ❌ Пентатоника как отдельный лад (минорная/блюзовая) — в MVP берётся подмножество `[1,2,3,5,6]` текущего лада.
- ❌ Ноты на нотоносце и визуализация на клавиатуре/грифе.
- ❌ Звено секвенции > 1 такта (паттерн всегда укладывается в один такт).

## 6. Риски и допущения

| ID | Риск | Решение |
| --- | --- | --- |
| R1 | Паттерн может выходить за пределы 7 ступеней (например `1531` берёт ступень 8) | Используем циклическое индексирование по `SCALE_INTERVALS` с переносом в следующую октаву (`(tonicPc + interval + 12) % 12`). |
| R2 | Пентатоника `[0,1,2,4,5]` берётся из текущего лада, но не во всех ладах есть «чистая» пентатоника | Для MVP паттерн берётся как подмножество ступеней текущего лада. Для мажора/минора/дориана это звучит корректно; для лидийского/локрийского — терпимо. Отдельные пентатонные лады — P2. |
| R3 | При множестве стартовых ступеней и `over-chords` прогрессия может стать слишком длинной | В режиме `over-chords` стартовая ступень назначается циклически (`i % startDegrees.length`), а не умножается на число ступеней. |
| R4 | `resolveChordTonePitchClass` для аккордовых тонов (1/3/5/7) уже есть, но паттерны требуют «ладовых» ступеней, а не аккордовых | В `over-chords` режиме стартовая pc берётся как `resolveChordTonePitchClass(symbol, startDegree, scaleType)`, затем паттерн строится диатонически от неё — тот же подход, что в опеваниях для целевой ступени. |
| R5 | Карточка может быть перегружена (паттерн из 5 нот + аккорд + ступень) | Компактный 3-строчный лейаут; ноты паттерна в одну строку через пробел; на мобильных форсировать `mode='current'`. |
| R6 | Размещение `sequences.ts` в `music-core` добавляет зависимость | `music-core` зависит только от stdlib + `@jazz/shared`; новые функции не нарушают boundaries. |

## 7. Метрики успеха

- Плитка «Секвенции» активна и не показывает «скоро».
- Пользователь проходит wizard за <60 секунд и запускает упражнение.
- Карточка отображает 3 строки: `Ступень N` → аккорд → ноты паттерна (корень выделен).
- `unified`-режим: на тоническом аккорде строятся паттерны для каждой выбранной стартовой ступени.
- `over-chords`-режим: на каждый аккорд — один такт с паттерном от циклической стартовой ступени.
- `SequenceType === 'all'` — тип паттерна меняется случайно на каждый такт.
- `direction: 'both'` — паттерн проигрывается вверх по ступеням, затем вниз.
- `playRandomly` — случайная тональность на каждый цикл прогрессии.
- Аккомпанемент, метроном, счётчик тактов работают корректно.
- Настройки сохраняются в `UserSettingsDTO`.
- `typecheck` + `lint` (boundaries) + `test` — зелёные.
- Существующие тесты `scaleExercise`, `enclosureExercise` и `chordExercise` не затронуты.
- Новые тесты `music-core/sequences.test.ts` покрывают все 11 паттернов.
- Новые тесты `practice-cards/sequenceExercise.test.ts` покрывают `unified`, `over-chords`, `all`, `playRandomly`, `direction`.
