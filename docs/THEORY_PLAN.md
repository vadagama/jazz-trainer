##### T-007. `OSMDSheetMusic` — нотный стан с подсветкой

- **Родительская функция:** 5.1.2 OSMD-интеграция
- **Приоритет:** P0
- **Сложность:** L (1–2w)
- **Слой:** ui
- **Плагин / Модуль:** `packages/ui/src/OSMDSheetMusic.tsx`
- **Описание:**
  Реализовать согласно THEORY_ARCHITECTURE.md §4.3:
  1. Props: `notes: string` (MusicXML или ABC-подобный формат), `playback: PlaybackConfig`
  2. Рендеринг: OSMD-нотоносец через `opensheetmusicdisplay`
  3. При `playback.highlightNotes = true`:
     - `TransportEngine.schedule()` — планирование нот
     - Подписка на события `TransportEngine`
     - Подсветка текущей ноты через OSMD API (cursor + color)
  4. Синтез звука: через `tone-audio-adapter` (AudioPort)
  5. Интеграционный тест: `OSMDSheetMusic` + `TransportEngine` — синхронизация подсветки (timing ±20ms)
- **Критерий готовности (DoD):** Ноты рендерятся на стане; при проигрывании подсвечивается текущая нота; тест синхронизации проходит
- **Зависит от задач:** T-001, T-004, T-005
- **Статус:** 🔴 Запланировано

**Риск:** OSMD может не справиться с джазовой нотацией (аккорды, альтерации). Митигация: проверить на прототипе до полной интеграции. Запасной вариант: VexFlow.

## 4. Критические пути

1. **T-007 (OSMDSheetMusic)** — самая рискованная задача. Если OSMD не справляется с джазовой нотацией, потребуется замена на VexFlow (доп. 1–2w). Блокирует T-010 и все лекции.

2. **T-005 (типы SDK)** — блокирует все UI-компоненты и лекции. Должна быть выполнена первой в Фазе 1.

3. **T-010 (LectureRenderer)** — блокирует создание всех 18 лекций. Критично для демонстрации ценности раздела.

4. **T-042 → T-043 → T-045 (прогресс API)** — цепочка блокирует всю геймификацию Фазы 3. Должна быть начата сразу при переходе к Фазе 3.

5. **T-050 (boss-battles)** — крупнейшая задача (XL), зависит от многих компонентов. Может быть распараллелена: начать проектирование на Фазе 2, реализацию — на Фазе 3.
