# MIDI INSTRUMENT VISION — Живой MIDI-инструмент и игра поверх аккомпанемента

**Дата:** 2026-06-16
**Горизонт:** 1 версия (после Piano/Rhodes, ≈4–6 недель)
**Статус:** 🟡 Черновик

## 1. Резюме (Executive Summary)

Jazz Trainer получает возможность стать **полноценным инструментом для живой практики**: пользователь подключает MIDI-клавиатуру, выбирает солирующий тембр и играет поверх автоматического аккомпанемента (бас, барабаны, фортепиано, Rhodes). Это превращает тренажёр из «слушаю и оцениваю» в «играю с бэндом».

Ключевая ценность: **live-практика**, а не только упражнения на оценку. Пользователь запускает любую гармоническую сетку или упражнение, слышит аккомпанемент ритм-секции и играет свою партию — импровизирует, отрабатывает фразировку, артикуляцию, взаимодействие с ансамблем. Визуальная обратная связь (MIDI-индикатор, виртуальная клавиатура) даёт уверенность, что инструмент подключён и работает.

Jazz Trainer становится **единственным веб-тренажёром**, который объединяет:

- Полноценный автоаккомпанемент (Piano, Rhodes, Bass, Drums) с живыми сэмплами
- Живой MIDI-инструмент с выбором солирующих тембров
- Визуализацию игры в реальном времени
- Упражнения и сетки — всё в одном интерфейсе, без установки

## 3. Новые функции (Features)

### 3.1. MIDI-подключение: автоопределение и настройка

- **Проблема:** Подключение MIDI-устройства сейчас требует ручного вызова `init()` на `WebMidiAdapter`. Пользователь не видит статус подключения, не может выбрать конкретное устройство из нескольких, не может настроить MIDI-канал.
- **Описание:**
  - **Автоопределение:** при загрузке приложения (или по кнопке «Connect MIDI») Jazz Trainer запрашивает доступ к MIDI-устройствам через `navigator.requestMIDIAccess()`. Если доступ получен — сканирует входы, выбирает первый активный. Если позже подключается новое устройство — автоматически переключается (с уведомлением).
  - **Ручной выбор:** в настройках (`/settings`) или в панели плеера — выпадающий список доступных MIDI-устройств. Пользователь может выбрать конкретный вход (если их несколько — клавиатура + пэды + контроллер).
  - **Выбор MIDI-канала:** для каждого устройства — настройка канала (0–15, по умолчанию `all` — принимать все каналы). Полезно для клавиатур с возможностью переключения каналов на лету.
  - **Индикатор подключения:** в панели плеера — небольшой светодиод:
    - 🔴 Красный — MIDI не подключён
    - 🟡 Жёлтый — MIDI доступен, но нет активных устройств
    - 🟢 Зелёный — устройство подключено. При каждой MIDI-ноте — короткая вспышка (мигание ~100ms).
  - **Hot-plug:** при подключении/отключении устройства во время сессии — плавная обработка без разрыва. Если активное устройство отключилось — авто-переключение на следующее доступное.
- **Конкуренты:** EarMaster имеет выбор MIDI-устройства и индикатор; Synthesia показывает подключение; **ни у кого нет авто-переключения между устройствами.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** новый плагин `play.midi-instrument` (или расширение `core-player`), адаптер `webmidi-adapter`
- **Зависимости:** `webmidi-adapter` (готов ✅). Требует доработки `InputPort` для поддержки выбора устройства по ID и фильтрации каналов.

### 3.2. Звуковой движок: синтез (Tone.js) и сэмплированные соло-инструменты

- **Проблема:** Сейчас MIDI-ввод только оценивается (`midiEval`), но не озвучивается. Пользователь не слышит свою игру через Jazz Trainer — звук идёт либо из внешнего синтезатора, либо не идёт вообще.
- **Описание:**

  **Уровень 1: Дефолтный синтезатор (Tone.js PolySynth)**
  - При подключении MIDI-устройства автоматически создаётся `Tone.PolySynth` (полифонический синтезатор на 16 голосов).
  - Дефолтный пресет: мягкий «electric piano» — `PolySynth` с 3 осцилляторами (sine, triangle, square с фильтром), лёгкий ревербератор.
  - Никаких сэмплов — мгновенная загрузка, работает всегда.
  - Параметры: attack (10ms), release (200ms), filter cutoff.

  **Уровень 2: Сэмплированные соло-инструменты**
  - Пользователь может выбрать солирующий тембр из каталога:

    | Инструмент                  | Источник сэмплов              | Тип звука                  | Размер        | Приоритет |
    | --------------------------- | ----------------------------- | -------------------------- | ------------- | --------- |
    | **Clarinet**                | VSCO Community / free samples | Деревянный духовой, тёплый | ~5 MB         | P1        |
    | **Vibraphone**              | Salamander/FreePats           | Металлический, джазовый    | ~4 MB         | P1        |
    | **Acoustic Guitar (nylon)** | Nylon Guitar samples          | Щипковый, мягкий           | ~3 MB         | P1        |
    | **Trumpet (muted)**         | VSCO Community                | Медный духовой             | ~3 MB         | P2        |
    | **Flute**                   | VSCO Community                | Деревянный духовой         | ~3 MB         | P2        |
    | **Acoustic Piano**          | Salamander Grand (уже есть!)  | Фортепиано                 | ~20 MB        | P1        |
    | **Rhodes**                  | jRhodes3c (уже есть!)         | Электро-пианино            | ~8 MB         | P1        |
    | **Synth Lead**              | Tone.js FMSynth               | Синтезаторный лид          | 0 MB (синтез) | P1        |

  - **Переиспользование существующих сэмплов:** Salamander Grand Piano и jRhodes3c уже загружены для аккомпанемента — они же могут использоваться как соло-инструменты (тот же `Sampler`, другой режим игры).
  - **Переключение тембра:** горячие клавиши или выпадающий список на панели плеера. Смена тембра мгновенная (для синтеза) или с короткой загрузкой (для сэмплов, lazy-load).

  **Архитектура:**
  - Новый интерфейс `SoloInstrument` в `music-core` (расширение существующего `Instrument` или отдельная абстракция):
    ```ts
    interface SoloInstrument {
      readonly id: string;
      readonly name: string;
      /** Called on MIDI note-on. */
      noteOn(midiNote: number, velocity: number, time?: number): void;
      /** Called on MIDI note-off. */
      noteOff(midiNote: number, time?: number): void;
      /** Connect to AudioPort output. */
      connect(destination: AudioNode): void;
      /** Release resources. */
      dispose(): void;
    }
    ```
  - Адаптер для Tone.js-синтеза: `SynthSoloInstrument implements SoloInstrument`
  - Адаптер для сэмплов: `SamplerSoloInstrument implements SoloInstrument`
  - `SoloInstrumentManifest` (аналог `InstrumentManifest` для соло-инструментов)

- **Конкуренты:** Soundtrap/BandLab дают множество синтезаторов и сэмплов, но без аккомпанемента. GarageBand — много тембров, но нет джазового бэнда. **Никто не даёт «выбрал тембр → играешь с бэндом» в браузере.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `music-core` (SoloInstrument, SynthSoloInstrument, SamplerSoloInstrument, soloInstrumentManifest)
- **Зависимости:** 3.1 (MIDI-подключение), `ToneAudioAdapter` (готов ✅). Требует Web Audio API, который уже используется.

### 3.3. Виртуальная клавиатура и визуализация игры

- **Проблема:** Пользователь не видит обратную связь от своей игры. При игре на MIDI-клавиатуре нет подтверждения, что нажатия достигли приложения и какой именно звук воспроизводится.
- **Описание:**

  **3.3.1. MIDI-индикатор в панели плеера**
  - Компактный индикатор (светодиод ~12×12px) рядом с транспортом.
  - Мигает зелёным на каждую note-on, гаснет на note-off.
  - Длительность вспышки: ~100ms (чётко видно даже при быстрой игре).
  - При Poly Pressure / Aftertouch — цвет меняется на синий (отличие от обычной ноты).

  **3.3.2. Виртуальная клавиатура (Virtual Keyboard)**
  - Располагается над панелью плеера (или в отдельной закреплённой панели).
  - Компактное представление: 2–3 октавы (C3–C5 по умолчанию), горизонтальный скролл при необходимости.
  - Клавиши, соответствующие играемым нотам, **подсвечиваются**:
    - Note-on: клавиша становится синей (или акцентным цветом темы) с яркостью, пропорциональной velocity.
    - Note-off: клавиша плавно гаснет (CSS transition ~150ms).
    - Одновременно могут быть подсвечены несколько клавиш (полифония).
  - **Режимы отображения:**
    - `free` — все клавиши активны (по умолчанию)
    - `scale-highlight` — подсвечены только ноты текущего лада/гаммы (для практики в тональности)
    - `chord-highlight` — подсвечены ноты текущего аккорда сетки (для импровизации по аккордам)
  - **Отображение названий нот:** опционально — подписи C, D, E... на белых клавишах.

  **3.3.3. Визуализация нажатий на сетке (Chord Grid Highlight)**
  - При активной сетке — такт, в котором звучит текущий аккорд, подсвечивается мягкой рамкой.
  - Если пользователь играет ноты, принадлежащие текущему аккорду — аккорд на сетке получает зелёную обводку (в тональности).
  - Если играет ноты вне аккорда — жёлтую (tensions, passing tones).
  - Если играет ноты вне тональности — красную (outside playing — тоже ценно для джаза!).

  **3.3.4. MIDI-лог (опциональная панель)**
  - Сворачиваемая панель под виртуальной клавиатурой: показывает последние 10 сыгранных нот в формате `C4 (vel: 87) → D4 (vel: 64) → ...`.
  - Полезно для отладки и понимания, что именно ушло в звук.

- **Конкуренты:** Synthesia — виртуальная клавиатура с падающими нотами; Flowkey — подсветка клавиш при обучении; **никто не подсвечивает аккорды сетки в реальном времени.**
- **Приоритет:** 3.3.1 P0, 3.3.2 P1, 3.3.3 P2, 3.3.4 P3
- **Усиливает плагин / Новая категория:** новый плагин `visual.midi-keyboard` (виртуальная клавиатура + индикатор), расширение `core-player` (MIDI-индикатор)
- **Зависимости:** 3.1 (MIDI-подключение), 3.2 (звуковой движок). Для 3.3.3 — доступ к текущей позиции транспорта и ChordTimeline.

### 3.4. Игра поверх автоаккомпанемента

- **Проблема:** Сейчас MIDI-ввод используется только в упражнениях (ear-training, rhythm-drills) с активной оценкой. Пользователь не может просто запустить сетку и играть поверх — MIDI не озвучивается, нет режима «свободной игры».
- **Описание:**
  - **Режим «Free Play»:** при запуске любой сетки или упражнения пользователь может играть на MIDI-клавиатуре поверх аккомпанемента. Аккомпанемент (Bass, Drums, Piano, Rhodes) работает как обычно — глобальный стиль, темп, громкости.
  - **Микшер соло-инструмента:** на панели плеера появляется регулятор громкости соло-инструмента (`soloVolume`, 0–1) — независимо от громкости аккомпанемента.
  - **Выбор тембра на лету:** выпадающий список «Solo Instrument» на панели плеера: `Synth (Default)` / `Acoustic Piano` / `Rhodes` / `Clarinet` / `Vibraphone` / `Guitar` / ... Переключение без остановки воспроизведения.
  - **«Уступи дорогу» (Ducking):** опциональный режим — когда пользователь играет, громкость Piano/Rhodes в аккомпанементе автоматически снижается на ~6 dB. Это чтобы соло-инструмент был слышен в миксе. Включается/выключается чекбоксом `Auto-duck`.
  - **Синхронизация с сеткой:** виртуальная клавиатура подсвечивает ноты текущего аккорда, помогая пользователю ориентироваться в гармонии.
  - **Loop-режим для практики:** пользователь зацикливает 2–4 такта, играет соло поверх, экспериментирует с фразировкой. Транспортные контролы (play, pause, stop, loop) работают как обычно.

- **Конкуренты:** iReal Pro — только аккомпанемент, MIDI-ввода нет; GarageBand — можно записать поверх минуса, но нет джазового автоаккомпанемента и визуализации гармонии. **Это уникальный режим для Jazz Trainer.**
- **Приоритет:** P0 (MVP)
- **Усиливает плагин / Новая категория:** `core-player` (режим Free Play, микшер соло), `music-core` (SoloInstrument, ducking-компрессор)
- **Зависимости:** 3.1, 3.2, 3.3.

### 3.5. Запись MIDI-перформанса

- **Проблема:** Пользователь играет вдохновенное соло — и оно теряется. Нет возможности сохранить и прослушать свою игру.
- **Описание:**
  - **MIDI-рекордер:** кнопка `Record` на панели плеера. При активации — все MIDI-события (note-on, note-off, velocity) записываются в память с таймстемпами, синхронизированными с транспортом.
  - **Воспроизведение:** кнопка `Play Recording` — записанный перформанс проигрывается через выбранный соло-инструмент синхронно с сеткой. Можно слушать со стороны, анализировать.
  - **Экспорт:** MIDI-файл (.mid) — скачать записанное в стандартном формате.
  - **Ограничение:** одна запись в памяти (не полноценный многодорожечный рекордер — это out of scope). Запись живёт только в сессии, не сохраняется на сервер (MVP).
  - **Визуализация при воспроизведении:** виртуальная клавиатура показывает записанные ноты (серым/тусклым цветом), чтобы отличать от живой игры.

- **Конкуренты:** GarageBand/Logic — полноценная запись; Soundtrap — запись в браузере; **но ни у кого нет записи поверх джазового автоаккомпанемента.**
- **Приоритет:** P2 (усиливает)
- **Усиливает плагин / Новая категория:** `music-core` (MidiRecorder), UI в `core-player` (кнопки Record/Play Recording)
- **Зависимости:** 3.1, 3.2.

### 3.6. Режим «Оценка импровизации» (Jam Eval)

- **Проблема:** В свободной игре пользователь не получает обратной связи о том, насколько его соло соответствует гармонии. Для обучения нужна мягкая, не штрафующая оценка.
- **Описание:**
  - **Гармонический анализ в реальном времени:** Jazz Trainer анализирует ноты, которые играет пользователь, относительно текущего аккорда сетки:
    - 🟢 **Chord tones** (1, 3, 5, 7) — зелёная подсветка на виртуальной клавиатуре
    - 🟡 **Tensions** (9, 11, 13) — жёлтая
    - 🔵 **Chromatic approaches / passing tones** — синяя
    - ⚪ **Outside notes** — без подсветки
  - **Статистика по окончании:** после остановки транспорта — панель с аналитикой:
    - % chord tones / tensions / outside
    - Распределение по аккордам сетки (на каких аккордах играл «внутри», на каких «снаружи»)
    - Средняя velocity, диапазон использованных нот
  - **Режим отображения:** можно включить/выключить (`jamEval: on/off`). Когда выключен — чистая свободная игра.

- **Конкуренты:** **Никто не делает гармонический анализ импровизации в реальном времени.** Это сильная образовательная фича.
- **Приоритет:** P2 (усиливает)
- **Усиливает плагин / Новая категория:** `music-core` (JamEval — расширение `midiEval`), визуализация в `visual.midi-keyboard`
- **Зависимости:** 3.1, 3.3.2, ChordTimeline.

### 3.7. MIDI-выход: отправка аккомпанемента на внешние синтезаторы (MIDI Output)

- **Проблема:** Часть пользователей хочет слышать аккомпанемент через свои внешние синтезаторы/сэмплеры, а не через встроенный Tone.js.
- **Описание:**
  - **Выбор MIDI-выхода:** в настройках — выпадающий список доступных MIDI-выходов (аналогично выбору входа). Можно выбрать конкретный выход для каждого инструмента аккомпанемента: Bass → Ch.1, Piano → Ch.2, Drums → Ch.10, Rhodes → Ch.3.
  - **Local-off для соло-инструмента:** если MIDI-клавиатура подключена и как вход (для игры), и как выход (к ней подключён внешний синтезатор), нужно отключить локальный звук на клавиатуре (`localControl: false`), иначе будет удвоение звука.
  - **Маршрутизация нот:** каждая нота аккомпанемента преобразуется в MIDI-сообщение (уже реализовано в `WebMidiAdapter.scheduleNote`) и отправляется на выбранный выходной порт.
  - **Синхронизация MIDI Clock:** опционально — отправка MIDI Clock (F8) на выбранный выход для синхронизации внешних секвенсоров/драм-машин.

- **Конкуренты:** iReal Pro — MIDI-выход есть; EarMaster — нет; **Jazz Trainer может дать поканальную маршрутизацию — выбор куда идёт каждый инструмент.**
- **Приоритет:** P2 (для продвинутых пользователей)
- **Усиливает плагин / Новая категория:** `webmidi-adapter` (MIDI-выход), настройки в Settings
- **Зависимости:** `WebMidiAdapter` (MIDI-выход уже реализован ✅ — `scheduleNote`, `scheduleClick`). Требуется UI для настройки портов.

### 3.8. Пресеты сцены (Scene Presets)

- **Проблема:** Для разных типов практики пользователю удобно иметь преднастроенные комбинации: «только бас и барабаны + гитара соло», «полный бэнд + пианино соло», «баллада с кларнетом».
- **Описание:**
  - **Scene Preset** — сохранённый набор настроек:
    - Стиль (swing/bossa/...)
    - Темп
    - Инструменты аккомпанемента (какие включены, громкость каждого)
    - Соло-инструмент и его тембр
    - Режим виртуальной клавиатуры (scale-highlight / chord-highlight)
  - **Стандартные пресеты (5 шт.) в поставке:**
    1. «Full Band + Piano» — весь бэнд, соло-пианино, scale-highlight
    2. «Bass & Drums + Guitar» — ритм-секция, найлоновая гитара соло
    3. «Ballad + Clarinet» — балладный стиль, разреженный аккомпанемент, кларнет
    4. «Solo Piano Practice» — только аккомпанемент-пианино (без соло), chord-highlight
    5. «Synth Jam» — фанк, синтезаторный лид, полный бэнд
  - **Пользовательские пресеты:** сохраняются в `userSettings` → `scenePresets: []`. Кнопка `Save Current as Preset` на панели плеера.
  - **Быстрое переключение:** выпадающий список пресетов на панели плеера.

- **Конкуренты:** iReal Pro — нет пресетов сцены; GarageBand — патчи, но не для сцены обучения. **Jazz Trainer может сделать уникальную для обучения механику.**
- **Приоритет:** P3 (nice-to-have, после MVP)
- **Усиливает плагин / Новая категория:** DTO в `@jazz/shared`, сохранение в `apps/api`, UI в `core-player`
- **Зависимости:** 3.1, 3.2, 3.4, API для хранения настроек (готово ✅).

## 4. Улучшения существующих функций

### 4.1. Доработка `webmidi-adapter`: выбор устройства и канала

- `InputPort` расширяется методами:
  - `listInputs(): Promise<MidiDeviceInfo[]>` — список доступных входов с именами
  - `selectInput(deviceId: string): void` — переключение на конкретное устройство
  - `setChannelFilter(channel: number | 'all'): void` — фильтр MIDI-каналов
- `WebMidiAdapter` уже хранит массив `inputs`, обработчик `onstatechange` для hot-plug. Доработка: дать возможность выбрать конкретный вход по ID, а не слушать все сразу.

### 4.2. Доработка `ToneAudioAdapter`: поддержка живых синтезаторов

- Сейчас `ToneAudioAdapter.scheduleNote` — это заглушка (инструменты аккомпанемента сами подключаются через `Tone.Transport.schedule`).
- Для живого MIDI: соло-инструмент подключается напрямую к `Tone.Destination` (или к шине микшера), минуя `Tone.Transport.schedule`, так как живые ноты не планируются заранее.
- Добавляется **master-шина**: `SoloBus` → подключается к `Tone.Destination`, на неё микшируются все соло-инструменты. Громкость шины управляется регулятором `soloVolume`.

### 4.3. Доработка панели плеера (`core-player`)

- Добавляются новые элементы UI:
  - MIDI-индикатор (светодиод) — в левой части панели, рядом с транспортом
  - `Solo Instrument` — выпадающий список тембров
  - `Solo Volume` — слайдер громкости соло-инструмента
  - `Preset` — выпадающий список пресетов сцены (после 3.8)
  - `Record` / `Play Recording` — кнопки (после 3.5)
  - `Jam Eval` — чекбокс включения оценки (после 3.6)
- Все новые элементы скрываются (или показываются серыми), если MIDI не подключён.

### 4.4. Обновление `docs/` и создание `docs/MIDI.md`

- Новый документ `docs/MIDI.md` — описание MIDI-подсистемы: подключение, соло-инструменты, синтез vs сэмплы.
- Существующий `docs/ARCHITECTURE_BASE.md` — обновить секцию «Звук и MIDI»: добавить `SoloInstrument`, `SynthSoloInstrument`, `SamplerSoloInstrument`, виртуальную клавиатуру.

## 5. Не войдёт в эту версию (Out of Scope)

- **Многодорожечная запись (DAW-подобный рекордер)** — только одна дорожка соло.
- **MIDI-контроллеры (knobs, faders, mod wheel)** — только note-on/off, velocity, aftertouch.
- **Поддержка MIDI 2.0** — только MIDI 1.0.
- **Разделение клавиатуры (split):** левая рука — аккорды для управления аккомпанементом, правая — соло. Оставляем на будущее (требует серьёзной доработки гармонического анализа).
- **Автоматическая генерация минусовки по MIDI-вводу** (сыграл аккорд → Jazz Trainer подхватил и играет с тобой).
- **Редактор синтезаторных пресетов** — только выбор из готовых.
- **Визуализация нот на нотном стане** — только виртуальная клавиатура.
- **Интеграция с DAW как VST-плагин** — только браузер.
- **Сохранение записей на сервер** — только в памяти сессии (экспорт MIDI-файла локально).
- **Bluetooth MIDI** — только USB/кабельные MIDI-устройства (Web MIDI API ограничен браузерами).

## 6. Риски и допущения

| Риск                                                                                    | Решение                                                                                                                                         |
| --------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Низкая latency в браузере (Web Audio vs ASIO/CoreAudio)                                 | Tone.js использует Web Audio с низкой задержкой (~10ms). Не конкурирует с ASIO, но для практики приемлемо. Предусмотреть настройку buffer size. |
| Полифония PolySynth нагружает CPU (16 голосов × осцилляторы)                            | PolySynth с простыми осцилляторами (sine + triangle) лёгок. В будущем — WebAssembly-синтезатор для тяжёлых патчей.                              |
| Сэмплы для соло-инструментов увеличивают общий вес приложения                           | Lazy-load: сэмплы загружаются только при выборе инструмента. Первый выбранный сэмпл кешируется. Дефолтный синтез всегда доступен.               |
| MIDI-канал по умолчанию «all» ловит ноты со всех каналов — возможен двойной триггер     | Фильтр каналов в `WebMidiAdapter` с настройкой по умолчанию. Для multi-zone клавиатур — выбор конкретного канала.                               |
| Разные браузеры по-разному поддерживают Web MIDI (Chrome ✅, Firefox — флаг, Safari ❌) | Определение поддержки и вывод предупреждения. Для Safari/мобильных — рекомендация использовать Chrome.                                          |
| Виртуальная клавиатура может быть слишком громоздкой на мобильных                       | Компактный режим: 1.5 октавы, автоскейлинг. На мобильных — горизонтальная ориентация.                                                           |
| Сэмплы соло-инструментов трудно найти с правильным звуком                               | Использовать VSCO Community (лицензия CC0), FreePats, сэмплы из уже имеющихся библиотек (Salamander).                                           |
| Ducking-компрессор может звучать неестественно при быстрой игре                         | Мягкий knee, attack ~20ms, release ~300ms. Пользователь может регулировать глубину ducking или отключить.                                       |

## 7. Метрики успеха

- Пользователь подключает MIDI-клавиатуру через USB → индикатор загорается зелёным, виртуальная клавиатура реагирует на нажатия
- При игре на клавиатуре звук синтезатора слышен с задержкой <20ms от нажатия
- Пользователь переключает соло-тембр (Synth → Piano → Clarinet) — звук меняется без остановки воспроизведения
- Запущена сетка в стиле Swing — пользователь играет соло поверх аккомпанемента, оба звука слышны в миксе
- При включённом ducking — аккомпанемент становится тише во время игры пользователя и возвращается в паузах
- `typecheck` + `lint` + `test` — зелёные
- Виртуальная клавиатура реагирует на нажатия с видимой задержкой <50ms
- Все существующие тесты проходят (MIDI-фичи не ломают ear-training, rhythm-drills)

## 9. Рекомендуемый порядок реализации

```
Фаза A: Инфраструктура (2 недели)
  A1. Расширение InputPort + выбор устройства/канала в WebMidiAdapter
  A2. SoloInstrument интерфейс + SynthSoloInstrument (дефолтный синтез)
  A3. MIDI-индикатор в панели плеера
  → После A: можно подключить MIDI-клавиатуру и услышать дефолтный синтезатор

Фаза B: Виртуальная клавиатура (1–1.5 недели)
  B1. visual.midi-keyboard плагин: компонент + визуализация нажатий
  B2. Интеграция с core-player: клавиатура над панелью плеера
  → После B: нажатия видны на виртуальной клавиатуре

Фаза C: Соло-инструменты и игра с аккомпанементом (1.5–2 недели)
  C1. SamplerSoloInstrument + манифесты для Piano, Rhodes (переиспользование)
  C2. Сэмплы для Clarinet, Vibraphone, Guitar (загрузка + манифесты)
  C3. Микшер соло-инструмента + авто-дакинг
  → После C: можно играть соло с выбранным тембром поверх сетки

Фаза D: Запись и оценка (1–1.5 недели)
  D1. MidiRecorder + кнопки Record/Play в плеере
  D2. JamEval (гармонический анализ импровизации)
  → После D: можно записать и оценить свою игру

Фаза E: Пресеты и MIDI-выход (1 неделя)
  E1. Scene Presets (сохранение/загрузка)
  E2. MIDI-выход: настройка портов и каналов
  → Релиз
```

Суммарная оценка: **6–8 недель** одним разработчиком.
